## 検索体験

### 全文検索への移行

- **現状**: `ILIKE '%keyword%'` の部分一致のみ
- **問題**: サジェストチップ（「近いイベント」等）の意図と検索ロジックが乖離
- **改善案**:
  - Supabase `textSearch` に移行（`to_tsvector('japanese', ...)` 要検証）
  - カテゴリ×キーワードの複合フィルタ（チップタップで `category:event` 自動入力）
  - 検索履歴をAsyncStorageに5件保存、次回表示時にサジェスト

### 検索中のフィードバック

- **現状**: 300msデバウンス中にUI変化なし
- **改善案**: 入力中にSearchアイコンをインラインスピナーに切替（実装済み）。さらに「検索中...」テキストを薄く表示

### 検索結果なしのUX

- **現状**: StateView の汎用メッセージ
- **改善案**: 「"○○" の検索結果はありません。似たキーワード: △△」のようにサジェスト導線を表示

## データ永続化

### 保存ボタン（ブックマーク）のDB連携

- **現状**: `feed/[id].tsx` のブックマークが `useState(false)` のみ。画面離脱でリセット
- **改善案**:
  - `bookmarks` テーブル追加（`user_id`, `post_id`, `created_at`）
  - プロフィール画面に「保存済み」タブ追加
  - 保存数が多い投稿は「街の記憶」にも反映

### フォロー機能

- **現状**: ボタンは非表示（未実装のため削除済み）
- **改善案**:
  - `follows` テーブル追加（`follower_id`, `following_id`）
  - プロフィール画面にフォロー/フォロワー数を追加
  - フォロー通知はリテンション向上に最も効く

## パフォーマンス

### 日付フィルタのサーバー化

- **現状**: フィードで全投稿を取得してからJS側で `hoursAgo` フィルタ
- **改善案**: `created_at` のレンジクエリに移行。投稿量増加への備え

### ページネーション

- **現状**: `limit` 指定のみ、スクロール末尾で手動
- **改善案**: `useInfiniteQuery` + `onEndReached` で無限スクロール

### スケルトンスクリーン

- **現状**: 初回ロードはスピナーのみ
- **改善案**: カード形状のシマーアニメーション（`Reanimated` opacity ループ）を3〜5個表示。200ms以上かかる場合のみ表示してチラつき防止

## エラー・フィードバック

### エラー表示の統一

- **現状**: `Alert.alert()` と `StateView` が混在
- **改善案**:
  - ネットワーク/権限エラー → `StateView`（画面置換）
  - 操作エラー（いいね失敗等） → トースト通知（画面上部スライドイン、3秒消去）
  - 認証切れ → `Alert.alert()` + ログインへ遷移
  - グローバルトーストプロバイダーを `_layout.tsx` に配置

### プルリフレッシュ成功時のハプティクス

- **現状**: リフレッシュ成功時のフィードバックなし
- **改善案**: `expo-haptics` の `ImpactFeedbackStyle.Light` を全画面に追加

## アクセシビリティ

### タップ領域44pt保証

- **現状**: TalkItemのいいね・返信アイコンが16px + パディングなし
- **改善案**: 全インタラクティブ要素に `minWidth: 44, minHeight: 44` を保証

### 残りのaccessibilityLabel

- **現状**: 主要要素には追加済みだが、TalkItem・設定画面・プロフィール画面は未対応
- **改善案**: 全タップ可能要素にラベル追加

## 通知

### プッシュ通知設計

- **対象イベント候補**:
  - 自分の投稿にいいねがついた
  - 自分のトークに返信がついた
  - フォローされた（フォロー機能実装後）
  - 近くに新しい投稿がある（位置情報ベース）
- **技術**: `expo-notifications` + FCM（Android）/ APNs（iOS）
- **UI**: 設定画面にカテゴリ別ON/OFFトグル

## 投稿画面の磨き込み

### プレースホルダーの具体化

- **現状**: タイトル・詳細のプレースホルダーが汎用的
- **改善案**: カテゴリ選択後にカテゴリ別の例文を表示
  - lifeline: 「例: ○○スーパー、本日お米入荷」
  - event: 「例: 土曜朝市 8:00〜 ○○広場」
  - help: 「例: 引越し手伝い募集（○日午前）」

### 詳細入力のドラフトチップ

- **改善案**: 詳細フィールドの上にワンタップで挿入できるフレーズチップ
  - 「先着順」「予約不要」「雨天中止」「駐車場あり」等

---

## トーク画面の磨き込み

### 文字数表示の言い換え

- **現状**: 「○○ / 140」
- **改善案**: 「あと ○○ 文字」（残り表記の方が心理的に入力しやすい）

### 投稿完了フィードバック

- **現状**: 成功後に画面が戻るだけ
- **改善案**: 成功時にハプティクス + トーストで「投稿しました」を一瞬表示

## 実装優先度まとめ

| 優先度 | 項目 | 理由 |
|--------|------|------|
| **最高** | Realtime購読移動 | バッジが動かないバグ |
| **最高** | 保存ボタンDB連携 | ユーザーの期待を裏切る |
| **高** | エラー表示統一 | アプリの信頼感 |
| **高** | 全文検索 | コア機能の品質 |
| **中** | 通知 | リテンションに直結 |
| **中** | ページネーション + サーバーフィルタ | スケーラビリティ |
| **中** | スケルトンスクリーン | 体感速度 |
| **低** | 投稿/トーク画面の磨き込み | テキスト変更が中心 |
| **低** | フォロー機能 | DB設計から必要 |
