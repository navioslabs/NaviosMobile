## 1. 方針

NaviOs は「地域情報をすばやく見る」アプリである。
アニメーションは **意味のある箇所だけ** に絞り、常時動く背景や無限ループ演出は原則使わない。

技術選定:
- **React Native Reanimated ~4.1.1** の `entering` / `exiting` / `layout` アニメーションを中心に使う
- 無限ループは LIVE・HOT など「リアルタイム性を示すバッジ」に限定する
- 追加パッケージは不要（既存の依存のみで実装済み）

## 3. 各実装の詳細

### 3.1 FAB 開閉アニメーション（完了）

**変更ファイル**: `components/layout/Fab.tsx`

**実装内容**:

1. **Backdrop**: `AnimatedPressable` に `FadeIn.duration(200)` / `FadeOut.duration(150)` を適用
2. **メニュー2枚**: `FadeInUp` でスタッガー表示（トーク: 0ms、投稿: 80ms 遅延）、閉じ時は `FadeOutDown`
3. **FAB アイコン**: `Plus` アイコン1つに統一し、`useSharedValue` + `withSpring` で 0deg ↔ 135deg 回転（開くと ✕ に見える）
4. **FAB ボタン本体**: タップ時に `withSpring(0.92)` → `withSpring(1)` でバウンス感

**spring パラメータ**: `damping: 14, stiffness: 150`

**設計判断**:
- `PenLine` / `Plus` の2アイコン切替をやめ、`Plus` 1つの回転で統一。アニメーションが連続的になる
- `Animated.createAnimatedComponent(Pressable)` で Backdrop を Animated 化
- `zIndex: 48, 52` の既存構造を維持

### 3.2 Nearby カード出現アニメーション（完了）

**変更ファイル**:
- `components/features/nearby/NearbyPostItem.tsx`
- `app/(tabs)/index.tsx`（`index` prop 追加）

**実装内容**:

- `NearbyPostItemProps` に `index?: number` を追加
- 通常アイテム: `Animated.View` でラップし `FadeInUp.delay(index * 60).duration(400).springify().damping(14)`
- featured カード: `FadeInUp.duration(400).springify().damping(14)`（遅延なし、先に表示）
- ヘルパー関数 `staggerDelay(i)` で最大 400ms にキャップ

**設計判断**:
- `Animated.View` で外側をラップする方式を採用（`Pressable` の `style` 関数を壊さないため）
- `app/(tabs)/index.tsx` の `SectionList.renderItem` から `index` を渡す

### 3.3 ScanHeader の LIVE / HOT 脈動（完了）

**変更ファイル**: `components/features/nearby/ScanHeader.tsx`

**実装内容**:

- `usePulse(periodMs)` カスタム hook を追加
- `withRepeat` + `withSequence` + `withTiming` で scale 1.0 ↔ 1.08 のゆるい脈動
- opacity も連動: `0.85 + (scale - 1.0) * 1.875`（scale 1.0 → opacity 0.85、scale 1.08 → opacity 1.0）
- LIVE バッジ: 周期 1600ms、HOT バッジ: 周期 1400ms（機械的に見えないようずらす）

**設計判断**:
- hook に切り出して LIVE と HOT で再利用
- `Easing.inOut(Easing.ease)` で自然な緩急
- UI スレッド実行のため JS スレッドへの負荷はゼロ

### 3.4 AI 検索の状態遷移アニメーション（完了）

**変更ファイル**:
- `app/(tabs)/ai.tsx`
- `components/features/ai/SuggestionChips.tsx`

**実装内容**:

1. **マイク / クリアボタン**: `Animated.View` で包み `FadeIn.duration(200)` / `FadeOut.duration(150)` でクロスフェード
2. **サジェストチップ**: 各チップを `FadeInRight.delay(i * 50).duration(300).springify().damping(14)` でスタッガー表示
3. **検索結果カード**: `FadeInUp.delay(Math.min(i * 60, 400)).duration(350)` で順次出現
4. **ローディング状態**: `ActivityIndicator` 領域を `Animated.View` + `FadeIn` / `FadeOut` でフェード
5. **おすすめフィード**: 同様に `FadeInUp` スタッガーを適用
6. **検索結果ヘッダー**: `Animated.Text` + `FadeIn.duration(300)` でフェードイン

## 4. やらないこと

以下は現時点では **非推奨** とする。

| 演出 | 理由 |
|------|------|
| 常時動く大きな背景アニメーション | 情報閲覧の妨げになる。NaviOs は「すばやく見る」文脈 |
| すべてのカードに無限ループ演出 | 動きすぎると読みにくさが先に来る |
| 画面遷移のカスタムトランジション | expo-router のデフォルトで十分。コスト対効果が低い |

## 5. 共通の実装ガイドライン

### アニメーションの数値基準（実装で使用した値）

| パラメータ | 値 | 使用箇所 |
|-----------|-----|---------|
| `entering` の `duration` | 200〜400ms | 全般 |
| スタッガー遅延 | 50〜80ms / item（最大 400ms） | Nearby カード、検索結果、サジェスト |
| `spring` の `damping` | 14 | FAB、Nearby、サジェスト、タブアイコン |
| `spring` の `stiffness` | 150 | FAB アイコン回転、タブアイコン |
| 脈動の周期 | LIVE: 1600ms、HOT: 1400ms | ScanHeader |
| scale の変化量 | 1.0〜1.08（脈動）、1.0〜1.12（タブ） | ScanHeader、タブアイコン |
| FAB バウンス | 1.0 → 0.92 → 1.0 | FAB |

### パフォーマンス注意

- Reanimated のアニメーションは UI スレッドで実行されるため JS スレッドへの負荷は低い
- `entering` を SectionList の全アイテムに付けているが、`staggerDelay` でキャップしているためレンダリング負荷は限定的
- 無限ループ（`usePulse`）は Reanimated の `withRepeat` を使用しており、コンポーネントのアンマウントで自動停止される

## 6. 今後の調整ポイント

実機テストで以下を確認・微調整することを推奨する。

- FAB メニューの `exiting` アニメーション: 条件レンダリングとの兼ね合いで端末によって挙動が異なる可能性がある
- Nearby SectionList の初回表示: アイテム数が多い場合のスタッガー体感速度
- Android でのタブアイコングロー: `elevation` で代用しているが、色付きグローにするには追加の背景 View が必要になる場合がある
- LIVE / HOT 脈動: 実機でのバッテリー消費への影響（微小だが長時間表示画面のため確認推奨）
