# 開発環境セットアップ

> 更新日: 2026-03-28

## 必要なバージョン

| ツール | バージョン |
|--------|-----------|
| Node.js | v24.14.0 |
| npm | 11.9.0 |
| Expo CLI | 54.0.23 |
| EAS CLI | >= 18.4.0 |

## 起動手順

### エミュレーター起動

1. Android Studio → Device Manager から仮想デバイスを起動
2. ホーム画面表示まで待機

### ビルド（初回 or ネイティブモジュール追加後）

```bash
npx expo run:android
```

### 開発サーバー（通常の開発）

```bash
npx expo start -c
```

### 実機開発

1. スマホとPCを同じWi-Fiに接続
2. `npx expo start` → QRコードをスキャン

## 環境変数

`.env` に以下を設定:

```
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_ANON_KEY=eyJxxx...
GOOGLE_MAPS_API_KEY=AIza...（任意）
```

## トラブルシューティング

| 問題 | 対処 |
|------|------|
| ホットリロードが反映されない | `npx expo start -c` でキャッシュクリア |
| iOS で古いバンドルが残る | `rm -rf node_modules/.cache` も併用 |
| 位置情報が取れない | エミュレーター: Extended Controls > Location で座標を設定 |
| TypeScript エラー確認 | `npx tsc --noEmit` |
| エミュレーターがフリーズ | `taskkill /F /IM qemu-system-x86_64.exe` → `adb kill-server` |

## iOS 開発の注意事項

| 注意点 | 理由 |
|--------|------|
| lucide アイコン名は v1.x でリネーム多数 | `icons.js` でエイリアス。追加時は `dist/esm/icons/` を確認 |
| Reanimated の shared value はレンダー中に書き込まない | `useEffect` 内で設定 |
| `usePreventRemove` のガード条件は state を使う | `useRef` だと再レンダーされない |
| フックは Provider の子孫で呼ぶ | `QueryClientProvider` の外で呼ぶとクラッシュ |
