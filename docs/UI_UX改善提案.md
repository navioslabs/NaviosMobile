# NaviOs UI/UX 最終修正提案書

> 対象: NaviOs 現行モック
> 更新日: 2026-03-25
> 位置づけ: UI改善フェーズの最終修正提案

---

## 1. 結論

UIの方向性はかなり整理されており、全体としては実装フェーズへ進める水準に近づいている。

ただし、最終的に揃えておきたい表現と導線の修正がまだ残っているため、本書では「最後に反映すべき修正提案」を明確化する。

特に以下の2点は必須修正とする。

- `ひとこと` は `トーク` に変更する
- `ホーム` は `フィード` に変更する

この2点は、タブ名、画面タイトル、関連文言を含めて統一的に反映する。

---

## 2. 必須修正項目

### 2.1 `ひとこと` を `トーク` に変更する

現在は `ひとこと` という名称になっているが、機能の実態としては「近くの声や会話の流れを読む / 投稿する」画面であり、単発の短文投稿だけを想起させる名称よりも `トーク` の方が用途と広がりに合っている。

修正対象:

- タブ名: `ひとこと` → `トーク`
- 画面タイトル: `ひとこと` → `トーク`
- 関連文言:
  - `最初のひとことを投稿してみましょう`
  - `ひとこと`
  - `近くの人にひとこと`

なども `トーク` 基準で見直す

期待効果:

- 投稿だけでなく会話性も含んだ画面として理解されやすくなる
- 画面の役割が狭く見えすぎる問題を避けられる
- 将来的なコメント、返信、スレッド化にも名称が耐えやすい

### 2.2 `ホーム` を `フィード` に変更する

現在の `ホーム` は、一般的なホーム画面というより、日付・カテゴリ・投稿一覧を扱う情報閲覧の中心画面になっている。そのため `フィード` の方が役割を正確に伝えられる。

修正対象:

- タブ名: `ホーム` → `フィード`
- 関連文言:
  - 画面内説明
  - ドキュメント上の画面名
  - AIへの入力データ

も `フィード` へ統一する

期待効果:

- 画面の中身と名称が一致する
- `ホーム` と `さがす` の役割差が見えやすくなる
- 実装・仕様書・会話の中で画面名を曖昧にせず扱える

## 3. 全体評価

### 良くなっている点

- タブ構成そのものは整理されている
- `さがす` 画面は検索中心の構造へ近づいている
- 空状態の扱いが改善されている
- 設定画面の実用性が上がっている
- 可読性改善が進んでいる
- 投稿時の入力制御が以前より自然になっている

### まだ最終調整したい点

- タブ名称の最終統一
- 画面間の役割差の見せ方
- 文字と行動導線の微調整
- 投稿完了体験の補強

## 4. 画面別の最終修正提案

### 4.1 フィード

現 `ホーム` は、最終的に `フィード` へ名称変更する。

修正提案:

- タブ名を `フィード` に変更
- `FeedSummary` の各指標は件数や条件が直感で伝わる文言へ調整
- 空状態に `別の日を見る` または `投稿する` の導線を追加
- 先頭カードが強調表示される理由ラベルを追加する

意図:

- 情報閲覧の中心画面として役割を明確にする
- 「この画面で何を見るか」を名前と内容で一致させる

### 4.2 トーク

現 `ひとこと` は、最終的に `トーク` へ名称変更する。

修正提案:

- タブ名とタイトルを `トーク` に変更
- 空状態メッセージも `トーク` 基準で自然な文言へ調整
- 一覧上部の簡易投稿導線は維持
- 可能であれば一覧デザインを少し会話寄りに寄せる

意図:

- 単発投稿よりも「地域の会話・空気感」を扱う画面として見せる
- 将来機能との整合性を確保する

### 4.3 さがす

`さがす` は方向性としてかなり良い。最終的には検索体験の完成度を上げたい。

修正提案:

- 検索結果のローディング状態を追加
- 結果0件時の代替提案をもう少し明確にする
- 検索前のおすすめ表示と検索後の結果表示の差をさらに整理する

意図:

- AI画面の価値を、雰囲気ではなく成果で見せる

### 4.4 ちかく

`ちかく` は方向性が明確で、近接情報画面として成立している。

修正提案:

- 徒歩時間など移動判断に変換した情報を追加
- `すぐ近く` セクションの強調をさらに高める
- `ScanHeader` に更新時刻など信頼補助情報を加える

意図:

- 投稿閲覧ではなく「今どこへ行くか」の判断画面に寄せる

### 4.5 設定

設定画面は実用性がかなり上がっているが、まだ最終調整の余地がある。

修正提案:

- `PremiumCard` の位置を再検討
- 各設定行に現在値表示を揃える
- 文字サイズ切替の見せ方を簡潔に保つ

意図:

- 設定画面としての一覧性を高める

### 4.6 投稿画面

投稿画面は制御が入ってきており、方向性は良い。

修正提案:

- 投稿完了時の軽い完了フィードバックを追加
- 必須項目や任意項目の説明を少し補う
- `トーク` 投稿画面の文言も名称変更に合わせて見直す

意図:

- 入力ストレスを減らし、安心して投稿できる体験にする

1. `ひとこと` を `トーク` に変更
2. `ホーム` を `フィード` に変更

4. フィード空状態の導線追加

### Middle

5. `ちかく` の移動判断情報追加
6. 設定画面の一覧性調整
7. 投稿完了体験の追加

### Low

8. トーク一覧デザインの会話寄り調整
9. `PremiumCard` の位置再調整
