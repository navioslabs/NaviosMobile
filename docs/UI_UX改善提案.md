## 1. 更新サマリー

前回の提案から、以下はすでに改善済みです。

- タブ名の日本語化: `ホーム / ひとこと / さがす / ちかく / 設定`
- `Talk` 画面の名称整理: `ひとこと` に統一
- `AI` 画面のヒーロー縮小
- `Talk` と `NearBy` の空状態改善
- `Settings` の冗長なステータスバー削除
- `Post` 画面でタイトル未入力時の投稿制御
- 一部 `Pressable` の押下フィードバック追加

このため、本書では「未着手の論点」と「次に効く改善」に絞って提案します。

## 2. 全体診断

### 現状の強み

- タブ名称が整理され、初見でも役割を推測しやすくなった
- 配色、余白、角丸がトークン化されており、画面の統一感が高い
- `ホーム` は日付、カテゴリ、サマリーで情報探索の切り口が明確
- `ちかく` は距離帯で分かれており、地域アプリらしい価値が伝わる
- `StateView` が導入され、空状態やエラー状態を共通化しやすい土台がある

### 現状の主要課題

| 課題 | 影響範囲 | 深刻度 |
|------|----------|--------|
| `ホーム / さがす / ちかく` の役割差がまだ近く、探索導線が重なっている | 全体IA | 高 |
| `さがす` が見た目は整った一方で、検索結果体験が未完成 | さがす | 高 |

## 3. 優先度の高い課題トップ5

### 1. `さがす` を「検索画面」として完結させる 【High】

**なぜ必要か**
現在の `さがす` は、検索欄、サジェスト、ランキング、クイックアクション、オートチェックインが同居しています。見た目は魅力的ですが、検索入力後の結果表示が弱く、「AIに相談する画面」としてまだ完結していません。

**どう変えるか**

- 画面構造を `検索欄 -> サジェスト -> 検索結果` に寄せる
- `query` 入力時は `PulseEventCard` を検索結果リストとして切り替え表示する
- 結果なしの場合は `StateView` で代替候補を案内する
- `QuickActions` は `ホーム` のショートカット機能へ寄せる
- `AIオートチェックイン` は `設定` に移動する

**期待効果**

- `さがす` を開いた時の一次行動が明確になる
- `ホーム` や `ちかく` との役割重複が減る
- AI機能の価値が「雰囲気」ではなく「結果」で伝わる

### 2. 可読性とタップしやすさの底上げ 【High】

**なぜ必要か**
現状の最小文字サイズは `10px`、補助テキストは `11px` が残っています。地域情報アプリは高齢者利用も強く想定されるため、現状のままだと読みづらさと誤タップのリスクがあります。

**どう変えるか**

- `FONT_SIZE.xxs` を `12`
- `FONT_SIZE.xs` を `12`
- `FONT_SIZE.sm` を `13`
- 本文基準 `FONT_SIZE.base` を `15` に引き上げる
- ボタンとチップの最小高さを `44px` 以上にそろえる
- `talk-post.tsx` の写真ボタン、`settings.tsx` のテーマトグル周辺、各カード内アイコン群のヒット領域を拡大する

**期待効果**

- 幅広い年齢層で読解しやすくなる
- タップミスが減り、使いやすさの体感が上がる

### 3. 操作フィードバックの共通化 【High】

**なぜ必要か**
一部の `Pressable` に `opacity` は入っていますが、全体ではまだ不統一です。押せる場所と押した結果が揃っていないため、画面によって操作品質に差が出ています。

**どう変えるか**

- `components/ui/` に共通の押下ラッパーを追加する
- `opacity` と軽い `scale` を標準化する
- 対象:
  - `FeedPostCard`
  - `NearbyPostItem`
  - `PulseEventCard`
  - `SettingsRow`
  - FABメニュー
  - 投稿画面の写真選択ボタン

**期待効果**

- アプリ全体の質感が揃う
- デジタル不慣れ層でも「押せた感」が伝わりやすくなる

### 4. `ホーム` と `ちかく` の役割差をさらに明確にする 【High】

**なぜ必要か**
タブ名は改善されたものの、どちらも最終的には投稿を一覧で見る画面です。ユーザーから見ると、「ホームでも近い投稿が見えるし、ちかくでも投稿が見える」状態が続いています。

**どう変えるか**

- `ホーム`: 時間軸とカテゴリ軸に特化
- `ちかく`: 距離軸と移動判断に特化
- `ちかく` に以下を追加
  - 徒歩時間
  - 現在地からの方向感ヒント
  - 地図導線またはミニマップ
- `ホーム` のサマリーから `近い順` を弱め、距離探索は `ちかく` に寄せる

**期待効果**

- タブ選択の迷いが減る
- 「いつ見るならホーム」「どこへ行くならちかく」という判断がしやすくなる

### 5. 投稿完了までの安心感を強める 【Middle】

**なぜ必要か**
`post.tsx` はタイトル未入力の制御が入りましたが、カテゴリは初期選択済みで、詳細入力や位置設定も任意です。`talk-post.tsx` も文字数表示はある一方で、超過時の明示や完了後の成功体験が弱いです。

**どう変えるか**

- 投稿ボタン押下前に必須項目の理由を明示する
- `talk-post.tsx` で `140文字超過時` は送信不可状態を見た目でも強調する
- 投稿完了時にトーストや軽い完了メッセージを出す
- 位置情報や写真が未設定でも投稿可能なら、その仕様を補助文で伝える

**期待効果**

- 投稿時の不安が減る
- 初回投稿率が上がる
- フォーム体験の雑味が減る

## 4. 画面ごとの改善案

### 4.1 ホーム

#### H-1: `FeedSummary` の意味を一目でわかる表現にする 【High】

現在のサマリーは便利ですが、数値やラベルの意味が直感で伝わらない可能性があります。

- `注目` は件数ベースにする
- `近く` は `200m以内` など条件をラベルに含める
- `まもなく終了` は残り時間の閾値を明示する

#### H-2: 先頭カードの特別感を説明付きで使う 【Middle】

先頭投稿を `featured` にしているため、なぜ目立っているのか補足があると理解しやすくなります。

- `おすすめ`
- `近くで話題`
- `締切が近い`

のような理由ラベルを追加する

#### H-3: 空状態に次の行動を加える 【Middle】

現在の空状態文言は改善されていますが、まだ受け身です。

- `別の日を見る`
- `投稿する`

のCTA追加で回遊を促せます

### 4.2 ひとこと

#### T-1: `ホーム` との差別化を見た目でも強める 【Middle】

タイトルは整理されましたが、一覧レイアウトがまだ標準的です。

- 吹き出し風
- テキスト主体
- 返信や共感が目立つ

というビジュアルに寄せると、情報の性質差が伝わります。

#### T-2: 初回投稿を後押しする導線を追加する 【Low】

空状態は整いましたが、一覧表示時にも投稿導線を感じやすくしたいです。

- ヘッダー右上に `投稿`
- 一覧上部に `今どう？` の軽い入力導線

を検討できます。

### 4.3 さがす

#### A-1: 検索入力時の表示切替を実装する 【High】

今は `「〇〇で検索...」` のプレビュー止まりです。ここを結果画面に変えるだけで、体験の完成度が大きく上がります。

- 入力前: サジェストとおすすめ
- 入力中: ローディング or 絞り込み中表示
- 結果あり: リスト
- 結果なし: 代替提案

#### A-2: `QuickActions` の配置を再検討する 【Middle】

価値はありますが、検索画面内だと主役を奪いやすいです。

- `ホーム` の上部ショートカットに移す
- `さがす` では折りたたみ表示にする

のどちらかが有効です。

#### A-3: オートチェックインの文脈を見直す 【Middle】

便利な機能ですが、検索画面にあると設定項目に見えます。

- `設定 > 通知`
- `設定 > AI機能`

へ移す方が自然です。

### 4.4 ちかく

#### N-1: 距離を「移動判断」に変換する 【High】

メートル表記だけでは、実際に行くかどうかの判断まで届きません。

- `徒歩2分`
- `徒歩6分`
- `自転車で5分`

のような行動換算を足すと、一気に使いやすくなります。

#### N-2: `すぐ近く` セクションを最重要情報として強化する 【Middle】

空状態表示は改善されています。次は、近いものがある時の強調です。

- 背景トーンを変える
- 先頭カードに距離バッジを強く出す
- `今行ける` の文言を追加する

#### N-3: スキャン体験の信頼感を増やす 【Low】

- 最終更新時刻
- 手動更新
- 位置取得状態

を `ScanHeader` に加えると、ライブ感が実用性に変わります。

### 4.5 設定

#### S-1: 文字サイズ設定を最優先で追加する 【High】

テーマ切替があるなら、文字サイズ切替も同等レベルで重要です。

- 標準
- 大
- 特大

の3段階だけでも価値があります。

#### S-2: `PremiumCard` の優先順位を下げる 【Middle】

設定を見に来たユーザーにとっては、今はやや早い位置にあります。

- 最下部に移動
- 小型カード化

のどちらかで、設定画面の主目的を邪魔しにくくできます。

#### S-3: 各設定行に現在値を見せる 【Low】

例:

- 通知設定: `ON`
- テーマ: `ダーク`
- 文字サイズ: `大`

と表示すると、一覧だけで状態把握できます。

### 4.6 投稿画面

#### P-1: `post.tsx` の入力補助を強化する 【Middle】

カテゴリが初期選択されているため、選んだつもりがないまま投稿する可能性があります。

- カテゴリ説明文を追加
- `この投稿は誰に役立つか` の補助文を追加
- 場所設定が任意か推奨かを明示する

#### P-2: `talk-post.tsx` の制限と状態を明確にする 【Middle】

- `140文字まで`
- `位置情報つきで投稿されます`
- 送信不可状態の視覚差

をより強く見せると安心して投稿できます。

## 6. 中長期で見直すべきUX改善

### 6.1 タブ構成の再検討

現在の5タブは以前より分かりやすくなりましたが、長期的には以下も検討できます。

- `ホーム`
- `ちかく`
- `さがす`
- `設定`

に絞り、`ひとこと` を `ホーム` 内セグメントへ統合する案です。

### 6.2 アクセシビリティ対応

- `accessibilityLabel`
- `accessibilityRole`
- `accessibilityHint`
- Dynamic Type 対応

は、今後の品質向上で避けて通れません。

### 6.3 実データ接続後の状態設計

モックデータ段階では見えにくいですが、実データ接続後は以下が重要になります。

- 読み込み中
- 通信失敗
- 検索0件
- 位置取得失敗
- 新着反映
