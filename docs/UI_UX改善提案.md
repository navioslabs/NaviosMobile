# NaviOs UI/UX 改善提案書

> 対象: NaviOs v1.0（モックアップ段階）
> 作成日: 2025-03-25
> 分析対象: Feed / Talk / AI / NearBy / Settings 全5タブ + 投稿モーダル2種

---

## 1. 全体診断

### 現状の強み

- **デザイントークンの統一度が高い**: 色・フォント・スペーシングが `lib/styles.ts` + `constants/theme.ts` に集約されており、視覚的な一貫性がある
- **Dark/Light テーマ対応済み**: 全画面でテーマ切替が機能する設計
- **カテゴリの視覚コード**: 色 + アイコンの組み合わせで、色覚障害にも配慮した分類表現
- **距離帯による情報の段階化**（NearBy）: 200m / 500m / その他 の3段階で直感的

### 現状の課題

| 課題 | 影響範囲 | 深刻度 |
|------|----------|--------|
| Feed / AI / NearBy の3タブで「投稿一覧」が重複し、役割の違いが分かりにくい | 全体IA | 高 |
| AI タブが「検索 + スコア + おすすめ + クイックアクション + チェックイン」と機能過多 | AI画面 | 高 |
| Talk と Feed の使い分けが曖昧（どちらも「地域の情報を見る」画面） | 全体IA | 中 |
| フォントサイズ最小10px — 高齢者・小画面端末で読みにくい | 全画面 | 中 |
| タップフィードバックが未実装（Pressable のデフォルトのみ） | 全画面 | 中 |
| アクセシビリティラベル未設定（accessibilityLabel / accessibilityRole） | 全画面 | 中 |
| 空状態（Empty State）が Feed 以外未実装 | Talk / AI / NearBy | 低 |
| StateView コンポーネントが存在するが未使用 | 全画面 | 低 |

## 2. 優先度の高い課題トップ5

### 1位: 情報設計の整理 — Feed / AI / NearBy の役割分離 【High】

**なぜ必要か**
現状、3タブとも「投稿の一覧を見る」画面になっており、ユーザーは「どこで何を見ればいいか」迷う。特に初回起動時、5タブの違いを理解するまでに離脱リスクがある。

**どう変えるか**

| タブ | 現状の役割 | 提案する役割 |
|------|-----------|-------------|
| **Feed** | 日付×カテゴリで投稿一覧 | **タイムライン**（時系列の全投稿。メインの閲覧画面） |
| **AI** | 検索＋スコア＋おすすめ＋クイックアクション | **検索＆発見**（検索に特化。QuickActions → Feed のフィルタショートカットに移動） |
| **NearBy** | 距離帯で投稿一覧 | **マップビュー**（地図中心の近接表示。リスト表示はサブ） |

- QuickActions（「今すぐ買える」「今日のイベント」等）を Feed の FeedSummary と統合
- PulseScoreCard は NearBy の ScanHeader に統合（「周辺の盛り上がり」は距離文脈と合致）
- AI タブは検索バー + サジェスト + 検索結果に絞る

**期待効果**
- 各タブの一次行動が明確になり、迷わず目的の画面に到達できる
- AI タブの情報過多が解消され、検索体験に集中できる

---

### 2位: 高齢者・デジタル不慣れ層への可読性改善 【High】

**なぜ必要か**
地域情報アプリは高齢者の利用率が高い。現状の最小フォント10px（バッジ）、11px（キャプション）は、老眼や小画面端末では判読困難。

**どう変えるか**

| 対象 | 現状 | 提案 |
|------|------|------|
| 最小フォント（バッジ） | 10px | **12px** |
| キャプション | 11px | **12px** |
| メタ情報 | 12px | **13px** |
| 本文テキスト | 14px | **15px**（既に lg で定義済み。base を 15px に引き上げ） |
| タッチターゲット最小サイズ | 約38px（一部ボタン） | **44px以上**（Apple HIG 準拠） |
| DatePicker の日付セル | 50×60px | **56×64px** |
| CategoryChips の高さ | 30px程度 | **36px以上** |

- `lib/styles.ts` の `FONT` 定数を一括調整するだけで全画面に反映される

**期待効果**
- WCAG AA のテキストサイズ基準を満たし、幅広い年齢層が快適に利用可能
- タップミスの減少 → 操作ストレス低減

---

### 3位: タップフィードバックの実装 【High】

**なぜ必要か**
現状、ほぼ全ての Pressable にフィードバックがない。ユーザーは「タップが認識されたか」を判断できず、連打や誤操作が発生する。特にデジタル不慣れ層で顕著。

**どう変えるか**
- 全 Pressable に `opacity: 0.7` のプレスフィードバックを統一適用
- カード系（FeedPostCard, NearbyPostItem, PulseEventCard）: `scale(0.98)` のアニメーション追加
- いいねボタン: ハートアニメーション（Reanimated で scale bounce）
- FAB: 押下時に軽い縮小（`scale(0.95)`）

**実装方法**: 共通の `PressableScale` コンポーネントを `components/ui/` に追加し、既存 Pressable を置換

**期待効果**
- 操作の確実性が向上し、UXの体感品質が大幅に改善

---

### 4位: Talk と Feed の差別化 【Middle】

**なぜ必要か**
Feed は「投稿カード」、Talk は「つぶやき」だが、どちらも「地域の情報を読む」画面。ユーザー視点では「2つのタイムラインがある」状態で混乱する。

**どう変えるか**

| 項目 | Feed | Talk |
|------|------|------|
| コンテンツ性質 | **公式・構造化情報**（イベント告知、物資配布、行政連絡） | **カジュアル・リアルタイム**（「今この店空いてる」「渋滞注意」） |
| 投稿形式 | タイトル＋詳細＋画像＋カテゴリ | 140文字テキスト＋画像（既に制限あり） |
| 視覚的差別化 | 画像中心のカードUI（現状維持） | テキスト中心のバブルUI（吹き出しデザインに変更） |
| タブ名 | 「Feed」のまま | **「ひとこと」に変更**（「Talk」「Chat」は1対1通話を連想させる） |
| タブアイコン | Rss（現状維持） | MessageCircle → **Megaphone** or **Radio**（放送・発信のニュアンス） |

**期待効果**
- 2画面の性質の違いが視覚的に明確になり、投稿先の判断も容易になる

---

### 5位: AI タブの一次行動の明確化 【High】

**なぜ必要か**
現状の AI タブは、ユーザーが画面を開いた瞬間「何をすればいいか」が分からない。ロゴ → 検索 → サジェスト → スコア → おすすめ → クイックアクション → チェックイン と縦に7セクション並び、視線が定まらない。

**どう変えるか**

```
変更前（現状）:
  ロゴ → 検索バー → サジェスト → PulseScore → PulseEventCards → QuickActions → AutoCheckIn

変更後:
  検索バー（最上部固定）→ サジェストチップ → 検索結果 or おすすめ
```

- ロゴセクション（68px のアイコン＋タイトル＋サブタイトル）を削除し、ヘッダーに「AI」アイコンだけ残す
- 検索バーを画面最上部に固定（スクロールしても常に表示）
- PulseScoreCard → NearBy の ScanHeader に移動
- QuickActions → Feed の FeedSummary に統合
- AutoCheckIn → Settings に移動
- 画面は「検索入力 → 結果表示」のシンプルな構造に

**期待効果**
- 「開いたら検索する」という一次行動が明確に
- 画面の役割が「AIに聞く」に集約され、他タブとの重複が解消

## 3. 画面ごとの改善案

### 3.1 Feed 画面

#### F-1: FeedSummary の文言改善 【Middle】

**なぜ**: 「今日の注目」「近くで空いている」「まもなく終了」の3ブロックは良い切り口だが、数値の意味が分かりにくい（例: 「92%」が何のスコアか不明）

**どう変えるか**:
- 「今日の注目」→ 「🔥 注目 3件」（件数を明示）
- 「近くで空いている」→ 「📍 200m以内 5件」
- 「まもなく終了」→ 「⏰ 1時間以内に終了 2件」
- ブロックサイズを 150×140px → 120×100px に縮小（スクロールなしで3つ見える）

**期待効果**: 一目で数値の意味が分かり、タップ動機が明確に

#### F-2: DatePicker の「今日」強調 【Low】

**なぜ**: 14日分の日付が並ぶが、「今日」がアクセントカラー以外で特別扱いされていない

**どう変えるか**:
- 「今日」セルにラベル「今日」を追加（日付の上に小文字で表示）
- アプリ起動時に自動スクロールで「今日」を中央に配置

**期待効果**: 高齢者でも「今日の情報を見ている」ことが即座に分かる

#### F-3: FeedPostCard の情報密度低減 【Middle】

**なぜ**: 1枚のカードに「ヘッダー（アバター＋名前＋カテゴリ＋時間）＋キャプション＋UrgencyBar＋CrowdTag＋CardActions＋距離バッジ」と6層の情報が重なっている。画像の上にオーバーレイで載せているため、背景画像次第で可読性が落ちる。

**どう変えるか**:
- CrowdTag と UrgencyBar は展開前は非表示、カードタップ後の詳細画面で表示
- または、カード下部をオーバーレイではなく不透明な背景エリアに変更（グラデーション依存をやめる）
- CardActions（いいね・コメント）はカード外に出す（カード下の独立行）

**期待効果**: カードの視認性向上、画像が暗くならずに済む

#### F-4: 空状態メッセージの改善 【Low】

**なぜ**: 「この日の投稿はまだありません」だけでは次の行動が分からない

**どう変えるか**:
- メッセージ下に「他の日を見る」ボタン or 「投稿する」ボタンを追加
- StateView コンポーネント（既存・未使用）を活用する

**期待効果**: 空状態からの離脱を防ぎ、回遊率向上

---

### 3.2 Talk 画面

#### T-1: 画面タイトルの再検討 【Middle】

**なぜ**: 「Talk」は英語で分かりにくく、「Chat」（タブアイコン名）との不一致もある。「近くのつぶやきをキャッチ」のサブタイトルでやっと意味が伝わる状態。

**どう変えるか**:
- タブ名: 「Chat」→「ひとこと」
- 画面タイトル: 「Talk」→「ひとこと」
- サブタイトル: 「近くのつぶやきをキャッチ」→ 「ご近所のリアルタイム情報」

**期待効果**: 日本語のみ対応アプリとして、全年齢層に意味が伝わる

#### T-2: TalkItem のアクションボタン拡大 【Low】

**なぜ**: いいね・コメント・シェアのアイコンが16pxで、高齢者にはタップしにくい

**どう変えるか**:
- アイコンサイズ: 16px → 20px
- タップエリア: 最低44×44pxを確保（padding で調整）

**期待効果**: 操作ミス減少

#### T-3: 投稿がない場合の空状態追加 【Low】

**なぜ**: Talk 画面は空状態が未実装。投稿ゼロだと真っ白な画面になる。

**どう変えるか**:
- StateView コンポーネントを使用
- メッセージ: 「まだつぶやきがありません」
- CTA: 「最初のひとことを投稿する」ボタン

**期待効果**: 新規ユーザーの初回投稿を促進

---

### 3.3 AI 画面

#### A-1: ロゴセクションの縮小 【High】

**なぜ**: 68px のロゴアイコン＋28px のタイトル＋サブタイトルで約160px の垂直スペースを消費。ファーストビューで検索バーが画面中央付近まで押し下げられる。

**どう変えるか**:
- ロゴアイコン: 68px → 40px
- タイトル: 28px → 20px
- サブタイトル: 削除
- または: ロゴセクション全体を削除し、検索バー内に小さなAIアイコンを配置

**期待効果**: 検索バーがファーストビューの上部に表示され、一次行動（検索）への到達が速くなる

#### A-2: サジェストチップの改善 【Middle】

**なぜ**: 「空いてるカフェ」「今近いイベント」等は良い例だが、横スクロールで一部が画面外に隠れる

**どう変えるか**:
- チップを2行に折り返し表示（FlexWrap）にして全て見える状態に
- または、よく使われる上位3つを大きなボタンで表示し、残りを「もっと見る」に

**期待効果**: サジェストの利用率向上（見えないチップはタップされない）

#### A-3: 検索結果の状態設計 【High】

**なぜ**: 現状、検索バーに文字を入力しても結果画面が存在しない（「〜で検索...」のテキストのみ）

**どう変えるか**:
- 検索中: ローディングアニメーション
- 結果あり: PulseEventCard のリストで表示
- 結果なし: 「見つかりませんでした。別のキーワードを試してください」+ サジェスト再表示
- StateView コンポーネントを活用

**期待効果**: 検索体験が完結し、AI タブの存在意義が確立

### 3.4 NearBy 画面

#### N-1: ScanHeader の情報追加 【Low】

**なぜ**: 「周辺をスキャン中」「LIVE」は雰囲気は良いが、実用情報が少ない

**どう変えるか**:
- 最終更新時刻を追加（「3分前に更新」）
- 手動リフレッシュボタンを追加
- PulseScoreCard の情報をここに統合（「盛り上がり度: 82/100」）

**期待効果**: リアルタイム性の信頼感向上

#### N-2: 距離セクションの色分け強化 【Low】

**なぜ**: 左のアクセントバー（3px）だけでは距離帯の違いが弱い

**どう変えるか**:
- セクションヘッダーの背景に薄い色を敷く（accent @ 5%, amber @ 5%）
- セクション間に 16px のスペースを追加

**期待効果**: 距離帯の視覚的区切りが明確になり、情報構造が理解しやすくなる

#### N-3: 「すぐ近く」セクションが空の場合 【Low】

**なぜ**: 200m以内に何もない場合、最も重要なセクションが表示されない

**どう変えるか**:
- 空の場合も「すぐ近く（200m以内）」セクションを表示
- メッセージ: 「200m以内にイベントはありません。少し足を伸ばしてみましょう」

**期待効果**: ユーザーが「何も検出されていない」のか「セクションが存在しない」のか区別できる

### 3.5 Settings 画面

#### S-1: フォントサイズ設定の追加 【High】

**なぜ**: 高齢者向けアプリとして、テーマ切替と同レベルの重要度でフォントサイズ調整が必要

**どう変えるか**:
- 「表示」セクションに「文字サイズ」行を追加
- 3段階: 標準 / 大 / 特大
- `lib/styles.ts` の `FONT` に倍率を適用する仕組み

**期待効果**: 高齢者の満足度・継続利用率が大幅向上

#### S-2: プレミアムカードの位置変更 【Low】

**なぜ**: 統計情報（チェックイン・バッジ）の直下にプレミアム誘導があり、設定項目を探すユーザーの視線を遮る

**どう変えるか**:
- プレミアムカードを画面最下部に移動
- または、プロフィールセクション内に小さなバッジとして配置

**期待効果**: 設定画面としての実用性が向上

#### S-3: ステータスサマリーバーの廃止 【Low】

**なぜ**: 「テーマ: Dark / 通知: ON / アカウント: ゲスト」の3項目が10pxフォントで並んでいるが、直下の設定項目と情報が重複しており冗長

**どう変えるか**: 削除。その分、設定項目自体に現在値を表示する（SettingsRow の右側にサブテキスト）

**期待効果**: 画面の情報密度低減、一覧性向上

### 3.6 投稿モーダル（Post / Talk-Post）

#### P-1: 投稿フォームのバリデーション追加 【High】

**なぜ**: 現状、タイトル空欄でも投稿ボタンが押せる状態。送信後のエラーハンドリングも未実装。

**どう変えるか**:
- 必須項目（カテゴリ、タイトル）が未入力の場合、投稿ボタンを非活性に
- 未入力項目にはインラインエラー表示（赤テキスト）
- Talk-Post の文字数制限（140文字）超過時のボタン非活性は実装済み — Post にも同様の仕組みを

**期待効果**: 不完全な投稿を防ぎ、データ品質を確保

#### P-2: カテゴリ選択の初期状態 【Low】

**なぜ**: カテゴリ未選択状態で投稿可能に見えるため、カテゴリなし投稿が発生しうる

**どう変えるか**:
- カテゴリセクションに「選択してください」の促しテキストを追加
- 未選択時は投稿ボタン非活性

**期待効果**: 投稿の分類精度が向上し、Feed のフィルタリングが正確に機能

### 3.7 グローバルナビゲーション

#### G-1: タブバーのラベル改善 【High】

**なぜ**: 「Feed」「Chat」「AI」「NearBy」「設定」のうち、英語3つ＋和英混在1つ＋日本語1つと統一感がない。日本語のみ対応アプリで英語ラベルは不親切。

**どう変えるか**:

| 現状 | 提案 |
|------|------|
| Feed | **フィード** or **ホーム** |
| Chat | **ひとこと** |
| AI | **さがす** |
| NearBy | **ちかく** |
| 設定 | 設定（現状維持） |

**期待効果**: 全年齢層がタブの意味を即座に理解できる

#### G-2: FAB メニューの文言改善 【Low】

**なぜ**: 「投稿する」と「つぶやく」の違いが、メニューの小さなサブテキスト（「イベント・物資・お知らせ」「近くの人にひとこと」）でしか説明されていない

**どう変えるか**:
- サブテキストのフォントサイズを 11px → 13px に拡大
- または、アイコンを大きく（20px → 28px）して視覚的な差別化を強化

**期待効果**: 投稿種別の選択ミス減少

## 5. 中長期で見直すべきUX改善

### 5.1 情報アーキテクチャの再設計（1〜2スプリント）

**現状の5タブ構成:**
```
Feed | Chat | AI | NearBy | 設定
```

**提案する4タブ構成:**
```
ホーム | ちかく | さがす | 設定
```

- **ホーム**: 現 Feed + Talk を統合。セグメントコントロールで「投稿」「ひとこと」を切替
- **ちかく**: 現 NearBy + PulseScore。地図ビュー追加
- **さがす**: 現 AI の検索機能に特化
- **設定**: 現状 + AutoCheckIn + 文字サイズ設定

**メリット**: タブ数減少で認知負荷低減。各タブの役割が1文で説明できる状態に。

### 5.2 アクセシビリティ対応の本格化（継続的）

- 全 Pressable に `accessibilityLabel` / `accessibilityRole` を設定
- `accessibilityHint` で操作結果を説明（「ダブルタップでいいねします」等）
- VoiceOver / TalkBack でのテスト実施
- 動的フォントサイズ（OS設定連動）の対応検討

### 5.3 オンボーディングの導入（1スプリント）

**なぜ**: 5タブの役割が初見で分かりにくいため、初回起動時に各タブの説明が必要

**提案**:
- 初回起動時に3〜4画面のウォークスルー
- 各タブの役割を1文 + スクリーンショットで説明
- スキップ可能にする（デジタル慣れしたユーザー向け）

### 5.4 リアルタイム性の体感向上（バックエンド連携後）

- Pull-to-Refresh の実装（全リスト画面）
- 新着投稿のインジケーター（「↑ 新しい投稿があります」バナー）
- NearBy の位置情報リアルタイム更新（バックグラウンド位置取得）

### 5.5 パフォーマンス最適化（ユーザー増加後）

- FlatList の無限スクロール / ページネーション実装
- 画像の遅延読み込み（expo-image のプレースホルダー活用）
- 水平スクロール（DatePicker, CategoryChips）の仮想化検討
