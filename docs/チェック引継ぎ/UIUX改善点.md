# NaviOs UI/UX 改善点

> 更新日: 2026-03-30
> 実機テスト + コードレビューに基づく指摘

---

## A. 高優先度（ユーザー体験に直結）

### A-1. 空状態が弱い
**現状**: `StateView` + Inbox アイコン + テキスト1行のみ
**問題**: 初回ユーザーが「何をすればいいか」わからず離脱する
**改善案**:
- 投稿なし → 「最初の投稿者になろう！」+ 投稿ボタン
- 検索結果なし → キーワードの代替候補を提案
- 通知なし → 「いいねやコメントが届くとここに表示されます」（実装済みだが、イラストがない）

### A-2. フィードモードの日付ピッカーが直感的でない
**現状**: 「今日」「明日」+ 数字の日付スクロール
**問題**: 「明日」のイベントを見る人は少ない。過去のイベント（昨日・一昨日）を見たい場合が多い
**改善案**:
- 「今日」「昨日」「今週」に変更
- または日付ピッカーをカレンダー形式に

### A-3. ちかくモード → フィードモードの切替が見落とされやすい
**現状**: 画面上部の小さなトグル（Radio / PenLine アイコン）
**問題**: ユーザーがフィードモードの存在に気づかない
**改善案**:
- 初回時に OnboardingTour でモード切替を説明
- トグルを大きくするか、スワイプで切替可能にする

### A-4. 投稿作成画面が長い
**現状**: `post.tsx` 465行。タイトル、本文、カテゴリ、期限、位置、画像を1画面で入力
**問題**: スクロール量が多く、入力途中で離脱しやすい
**改善案**:
- ステップ形式（1.カテゴリ選択 → 2.タイトル+本文 → 3.詳細設定）
- または最低限（タイトル+カテゴリ）で投稿可能にし、残りは任意

### A-5. いいねの手応えがない
**現状**: Heart アイコンの色が変わるだけ（`opacity: pressed ? 0.7 : 1`）
**問題**: SNSとして最重要のインタラクションなのにフィードバックが弱い
**改善案**:
- ハートのスケールアニメーション（RN組み込みAnimatedで実装可能）
- expo-haptics によるハプティクスフィードバック
- ※ Reanimated は React Compiler と競合するため、RN 組み込み Animated を使うこと

## B. 中優先度（磨き込み）

### B-1. 設定画面の折り紙テーマトグルが伝わりにくい
**現状**: SVG の折り紙鶴がダーク/ライトで変化する凝ったアニメーション
**問題**: 初見で「これがテーマ切替」と気づきにくい
**改善案**: トグルの横に「ダーク」「ライト」のテキストラベルを追加

### B-2. プロフィール画面のタブ（投稿/ひとこと）が見えにくい
**現状**: テキストのみのタブ。アクティブ時のアンダーラインが細い
**改善案**: アクティブタブにアクセントカラーの背景を付ける

### B-3. 通知画面にフィルタがない
**現状**: 全通知を時系列で一覧表示
**問題**: いいね通知が大量にあるとコメント通知が埋もれる
**改善案**: 「すべて」「いいね」「コメント」「返信」のフィルタタブ

### B-4. 投稿詳細画面の画像が小さい
**現状**: 固定サイズで表示。タップしても拡大不可
**改善案**: タップで全画面表示 + ピンチズーム

### B-5. 街の記憶画面のデータ表示
**現状**: 月ごとにグループ化されたリスト
**問題**: 地図上で可視化されないので「街」の感覚が薄い
**改善案**: 地図ビューオプションを追加（投稿位置にピンを立てる）

### B-6. ゴーストカウントダウンの視覚的重みが弱い
**現状**: 小さなテキストで「残り○時間」
**改善案**: 残り6時間以内でテキスト色をアンバーに、残り1時間で赤に変化

## C. 低優先度（リリース後）

### C-1. Pull-to-Refresh のフィードバックが標準的
**現状**: OS標準の RefreshControl
**改善案**: カスタムリフレッシュインジケーター（ロゴ回転等）
- ※ `PulseRefresh` は Reanimated 依存のため現在削除済み。RN Animated で再実装可能

### C-2. タブ切替にハプティクスがない
**現状**: タップしても触覚フィードバックなし
**改善案**: `Haptics.selectionAsync()` をタブ切替時に追加
- ※ 以前実装したが Reanimated エラーと同時期だったため revert。単独で再実装可能

### C-3. スケルトンがシンプル
**現状**: 単色パルスのみ（Reanimated 依存）
**改善案**: グラデーションシマー（左→右に光が走る）をRN Animated で実装

### C-4. オンボーディングの改善
**現状**: ステップ表示テキスト + ボタン
**改善案**: 各ステップにイラストやアニメーションを追加

### C-5. ダークモードのカード奥行き
**現状**: `surface` 系カラーが1段階のみ
**改善案**: `surface2`, `surface3`, `surface4` の段階を増やし、カードの階層感を出す

---

## D. 技術的改善

### D-1. Reanimated → RN Animated 移行
**現状**: 27ファイルで Reanimated 使用。React Compiler と競合
**対応**:
- ホーム画面周辺は移行済み（ScanHeader, FeaturedGlow, NearbyPostItem）
- 残り: FAB, Toast, Skeleton, SplashTransition, GuestLoginSheet, OnboardingTour 等
- `entering` / `exiting` は削除して即時表示にするか、RN Animated で代替

### D-2. 画面ファイルの行数削減
| ファイル | 現在 | 目標 |
|---|---|---|
| `post.tsx` | 465行 | 300行以下（ステップ分割 or コンポーネント抽出） |
| `ai.tsx` | 450行 | 350行以下（セクションをコンポーネント化） |
| `index.tsx` | 413行 | 350行以下（フィードモードを別ファイルに） |
| `feed/[id].tsx` | 401行 | 300行以下（コメントセクション抽出） |

### D-3. アクセシビリティ強化
- 装飾要素のアイコンボタンに `accessibilityLabel` 追加
- フォーカス順序の最適化
- スクリーンリーダーでの操作確認
