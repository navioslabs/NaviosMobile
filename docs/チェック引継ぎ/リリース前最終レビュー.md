## 1. プロジェクト概要

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- **アプリ名**: NaviOs（ナビオス）— 「近くを知る。暮らしが変わる。」
- **種別**: クロスプラットフォームモバイルアプリ（iOS / Android / Web）
- **技術スタック**: Expo ~54 + React Native 0.81 + TypeScript 5.9 + Supabase (Auth, Postgres, PostGIS, Storage) + React Query + Zustand + React Native Reanimated
- **対象ユーザー**: 地域住民（一般消費者向け）— 日本語のみ対応
- **主要機能**:
  - **ちかく（Nearby）**: GPS連動で距離別に周辺投稿を表示（200m / 500m / それ以上）
  - **タイムライン（Talk）**: 140文字のリアルタイムつぶやき（24時間で消える「ゴースト投稿」＋10いいねで殿堂入り）
  - **さがす（Search/Discover）**: トリグラム検索 + トレンド + 週間ランキング + AIサジェスチョン
  - **フィード（Feed）**: カテゴリ別・日付別の投稿一覧（lifeline / event / help）
  - **バッジシステム**: 投稿・いいね数に基づくエリアバッジ（住民 / 顔役 / レジェンド）

### 特に良い点（Good Parts）

1. **RLSポリシーの設計が優秀**: 全テーブルに適切なRow Level Securityが設定されており、バッジテーブルは直接操作を完全にブロックし`SECURITY DEFINER`関数経由のみに限定している。セキュリティの基盤として非常に堅牢。

2. **React Queryの運用が模範的**: queryKeyが機能単位で一貫しており、mutation成功時のキャッシュ無効化も適切。サーバーデータをZustandに持たないという方針が徹底されている。

3. **エラーUI設計が秀逸**: `StateView`コンポーネントでErrorKindを5種類に分類し、エラー種別ごとにアイコン・タイトル・ガイダンス・リカバリーアクションを提供。ユーザーが「何が起きたか」「どうすればいいか」を即座に理解できる設計。

4. **Skeletonローディングの完成度**: Reanimatedを使ったパルスアニメーション付きSkeletonが投稿カード・つぶやき・詳細画面それぞれに用意されており、体感的な読み込み速度を向上させている。

5. **入力バリデーションの二重防御**: Zodスキーマ（クライアント）とPostgreSQL CHECK制約（サーバー）の両方で同一ルールを適用。どちらか一方が突破されてもデータ整合性が保たれる。

6. **アニメーションの作り込み**: Reanimatedを50+箇所で活用し、FadeIn/SlideIn/Spring/パルス/interpolateColorなど多彩なアニメーションでネイティブ級のUXを実現。
