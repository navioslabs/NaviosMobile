# NaviOs を最高のアプリにするために — UI/UX 改修ロードマップ

> 作成日: 2026-03-30
> 対象: NaviOs v1.0 （Expo 54 + React Native 0.81）
> 目的: 「地域情報アプリ」から「毎日開きたくなるローカルSNS」への進化

---

## 1. ファーストインプレッション — 起動体験

### 1-1. スプラッシュ → ホームのシームレス遷移

| 現状 | 改修案 |
|------|--------|
| Expo デフォルトのスプラッシュ → いきなりホーム | スプラッシュのロゴが**そのままヘッダー位置にスライド**して着地する連続アニメーション |

```
実装イメージ:
SplashScreen.preventAutoHideAsync()
  → ロゴ中央表示（scale 1.5, opacity 1）
  → withTiming で headerの位置・サイズへ移動（600ms, Easing.bezier）
  → 背景を surface カラーにクロスフェード
  → SplashScreen.hideAsync()
  → ホーム画面が下からフェードイン
```

**効果**: 「アプリが一瞬で立ち上がった」感覚を演出。ブランド認知も強化。

### 1-2. オンボーディングの刷新

| 現状 | 改修案 |
|------|--------|
| OnboardingTour でステップ表示 | **Lottie アニメーション**付きの3ステップウォークスルー |

- Step 1: 地図上にピンがポンと立つアニメーション →「近くの情報がすぐ届く」
- Step 2: 投稿カードが次々流れるアニメーション →「みんなが発信、みんなが受信」
- Step 3: ハートが弾けるアニメーション →「いいねで地域がつながる」
- 各ステップ間は**スワイプ遷移**（PanGestureHandler + interpolate でパララックス）

## 2. ナビゲーション & タブバー

### 2-1. タブ切り替えアニメーション

| 現状 | 改修案 |
|------|--------|
| デフォルトの即時切り替え | **共有要素トランジション（Shared Element Transition）** |

```
実装方針:
- react-native-reanimated の sharedTransitionTag を活用
- ちかくタブ → 投稿詳細: カードがそのまま拡大して詳細画面になる
- タイムラインタブ → トーク詳細: バブルが展開するアニメーション
- タブ間移動: コンテンツが進行方向にスライド（左タブ→右タブ = 左から右へ）
```

### 2-2. タブバーのマイクロインタラクション

| 現状 | 改修案 |
|------|--------|
| AIタブのみスケールアニメーション | 全タブにアイコンアニメーション追加 |

```
各タブのアニメーション案:
- ちかく (Radio): タップ時に電波が広がるリング（scale 1→2, opacity 1→0）
- タイムライン (MessageCircle): タップ時にアイコンが軽くバウンス + 吹き出しが膨らむ
- さがす (Sparkles): 現状のスケール + キラキラパーティクルを2-3個散らす
- 設定 (Settings): タップ時にギアが90度回転（withSpring）

実装: withSequence + withSpring (damping: 10, stiffness: 200)
所要時間: 300-400ms、操作を阻害しない軽さに
```

### 2-3. FAB（フローティングアクションボタン）の改善

| 現状 | 改修案 |
|------|--------|
| 回転 + メニュー2項目のフェード | **形態変化 FAB** |

```
改修詳細:
- タップ: 円形 → 角丸長方形にモーフィング（width 64→280, withSpring）
- メニュー項目が内部に横並びで出現（stagger 80ms）
- 背景ブラー: BlurView (expo-blur) で背景をぼかす（intensity 20→40）
- 閉じる: 逆再生で元の円形に戻る
- 長押し: 直接「投稿する」画面へ（ショートカット）
```

## 3. カード & リスト表示

### 3-1. 投稿カードのリデザイン

| 現状 | 改修案 |
|------|--------|
| フラットなカードにテキスト + 画像 | **レイヤード・カードデザイン** |

```
デザイン改修:
- カード背景: surface にわずかなグラデーション（上: surface, 下: surface + 3%白）
- 画像: 角丸を16px→20pxに。画像なし時のプレースホルダーをより洗練
- カテゴリバッジ: 左上に半透明ピル（blur背景）で浮遊感
- 緊急度インジケーター:
  - 期限24h以内: カード左端に赤のアクセントライン（3px, animated pulse）
  - 期限3日以内: アンバーのアクセントライン
  - それ以外: アクセントカラーの細線（1px）
- いいねボタン: タップ時にハートが弾ける（scale 1→1.4→1, withSequence 300ms）
  + 小さなパーティクル3-5個が放射状に飛ぶ

影の改善:
- iOS: shadowOffset {0, 2}, shadowRadius 8→12, shadowOpacity 0.06→0.08
- Android: elevation 2→3
- ダークモード: border を glow カラーに（0.05 opacity）
```

### 3-2. リストのスクロール体験

| 現状 | 改修案 |
|------|--------|
| FlatList + FadeInUp スタッガー | **スクロール連動アニメーション** |

```
実装案:
- onScroll の scrollY を useSharedValue で追跡
- 各カードの表示領域に入った瞬間:
  - opacity: 0→1（200ms）
  - translateY: 30→0（withSpring, damping 18）
  - scale: 0.95→1（withSpring）
- スクロール時のヘッダー:
  - 上スクロール: ヘッダーが縮小（高さ 60→44）、ロゴのみ表示
  - 下スクロール: ヘッダーが復帰
  - withTiming + clamp で滑らかに
- Pull-to-refresh:
  - カスタムアニメーション: ロゴの「N」が回転（0→360deg）
  - 引っ張り量に応じて回転角度が interpolate
```

### 3-3. 空状態（Empty State）の改善

| 現状 | 改修案 |
|------|--------|
| Inbox アイコン + テキスト | **イラスト + アクション誘導** |

```
改修案:
- 状況別のイラスト or Lottie:
  - 投稿なし: 街のイラスト + 「最初の投稿者になろう！」
  - 検索結果なし: 虫眼鏡が首をかしげるアニメーション
  - 通知なし: ベルが静かに揺れるアニメーション
- CTA ボタンを追加:
  - 投稿なし → 「投稿する」ボタン（直接 post 画面へ）
  - 検索結果なし → 「条件を変えて探す」
- 背景にうっすらとパターン（ドット or 波線）を配置
```

## 4. 詳細画面の没入感

### 4-1. 投稿詳細画面

```
改修ポイント:

■ 画像ビューアー
- タップで全画面表示（SharedTransition でシームレスに拡大）
- ピンチズーム対応（GestureHandler の PinchGestureHandler）
- スワイプで画像切り替え + 閉じる（下スワイプでdismiss）
- 画像インジケーター: ドットをアクティブ時に伸長（width 6→18, withSpring）

■ パララックスヘッダー
- スクロールに連動して画像がゆっくり移動（parallax ratio 0.5）
- 一定量スクロールで画像がフェードアウト + タイトルがヘッダーに移動

■ コメント入力
- キーボード連動で入力欄がスムーズに上昇
- 送信時: コメントがリストに追加される際に FadeInUp アニメーション
- 送信ボタン: 入力中は withSpring で scale 0→1 出現

■ いいねアニメーション
- ダブルタップでいいね（画像上にハートが浮かび上がる、Instagram風）
- ハート: scale 0→1.2→1, opacity 1→0（800ms）
- 同時に Haptics.impactAsync(Medium)
```

### 4-2. トーク詳細画面

```
改修ポイント:

■ ゴーストカウントダウンの演出強化
- 残り6時間以内: 背景が徐々に透明化するアニメーション
- 残り1時間以内: テキストに微妙な揺れ（translateX ±1px, 2000ms周期）
- 消滅の瞬間: パーティクルが散って消える演出（もし見ている場合）

■ 殿堂入り演出
- 10いいね到達: 画面全体にゴールドのコンフェッティ
- トロフィーアイコンが回転しながら出現（3D flip）
- 「殿堂入り！」テキストが withSequence で拡大→縮小→定位置
```

## まとめ

NaviOs の現状は**機能としては十分に揃っている**。
次のステージに進むために必要なのは「**触っていて気持ちいい**」という体験の層を重ねること。

重要な原則:
1. **アニメーションは 300ms 以内** — それ以上は「遅い」と感じる
2. **すべての操作にフィードバック** — 視覚 + 触覚（ハプティクス）の二重フィードバック
3. **引き算のデザイン** — 装飾ではなく、情報の優先度を伝えるためにアニメーションを使う
4. **一貫性** — 同じ種類の操作は同じアニメーションパターンで（spring のパラメータを定数化）
5. **パフォーマンス最優先** — 60fps を下回るアニメーションは実装しない方がマシ
