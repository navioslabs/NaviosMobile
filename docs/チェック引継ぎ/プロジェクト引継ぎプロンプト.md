## 1. プロジェクト概要

- **名前**: NaviOs（ナビオス）— 地域情報共有モバイルアプリ
- **コンセプト**: 「すぐそばの暮らしが、見えてくる。」
- **ターゲット**: 日本の地域住民（日本語のみ）
- **スタック**: Expo 54 + React Native 0.81 + TypeScript 5.9
- **バックエンド**: Supabase（Auth / Postgres + PostGIS / Storage / Realtime）
- **状態管理**: React Query（サーバーデータ） + Zustand（UI状態5ストア）
- **UI**: StyleSheet + React Native Reanimated + Gesture Handler
- **フォーム**: React Hook Form + Zod
- **アイコン**: lucide-react-native（`lib/icons.js` で再エクスポート）
- **ルーティング**: Expo Router（ファイルベース）
- **エントリ**: `expo-router/entry`

## 2. 画面構成

```
Root (_layout.tsx) — QueryClientProvider + AuthProvider + ErrorBoundary
├─ (auth)
│  ├─ /login — メール・パスワードログイン
│  └─ /signup — サインアップ
│
└─ (tabs) — Header（ロゴ+通知+プロフィール） + FAB（投稿/つぶやき作成）
   ├─ / (index) — ホーム画面
   │  ├─ ちかくモード: GPS連動で距離別に投稿表示（300m/600m/それ以上）
   │  └─ フィードモード: カテゴリ×日付で投稿一覧
   ├─ /talk — タイムライン（140字つぶやき、24hで消えるゴースト投稿）
   ├─ /ai — さがす（トリグラム検索、話題のタグ、トレンド、週間ランキング）
   └─ /settings — 設定（テーマ、文字サイズ、プロフィール）

   スタック画面
   ├─ /post — 投稿作成（モーダル）
   ├─ /talk-post — つぶやき作成（モーダル）
   ├─ /feed/[id] — 投稿詳細（コメント、いいね、シェア、通報）
   ├─ /talk-detail/[id] — つぶやき詳細（返信、殿堂入り）
   ├─ /profile/[id] — ユーザープロフィール
   ├─ /profile/edit — プロフィール編集
   ├─ /notifications — 通知一覧
   └─ /street-history — 街の記憶
```

## 4. データフローパターン

```
画面 (app/) → hooks/ (React Query) → lib/ (Supabase呼び出し) → DB
                                    ↑
                              stores/ (UI状態のみ)
```

- **サーバーデータ**: 必ず React Query 経由。Zustand に保持しない
- **queryKey設計**: `["posts", "list"]`, `["posts", "nearby"]`, `["posts", "detail", id]` 等
- **mutation成功後**: `invalidateQueries` でキャッシュ更新
- **エラー**: lib/ で throw → hooks/画面で catch → `AppError` の `userMessage` を Toast 表示

## 5. 主要機能の仕組み

### 近隣投稿（ちかく）
- `useLocation({ watch: true })` でGPS監視
- 100m以上移動で `useNearbyPosts` が自動再取得
- DB側: `get_nearby_posts` RPC（PostGIS `ST_DWithin`）
- UI: 300m以内 / 600m以内 / それ以上 の3セクション

### ゴースト投稿（タイムライン）
- 投稿から24時間で自動削除（DB側で管理）
- 10いいねで殿堂入り → `is_hall_of_fame = true` → 永久保存
- トリガー: `on_like_check_hall_of_fame`

### 検索
- PostgreSQL `pg_trgm`（トリグラム）による部分一致検索
- 投稿/ひとこと タブ切替
- 話題のタグ: `get_trending_tags` RPC（位置ベース5km圏内、直近7日）

### バッジシステム
- 投稿数・いいね数に基づくエリアバッジ（住民/顔役/レジェンド）
- SECURITY DEFINER関数経由でのみ更新可能

### リアルタイム
- Supabase Realtime で talks テーブルを監視
- `useRealtimeTalks` hook で新着を検知 → badgeStore に未読カウント

## 6. テーマシステム

- `useThemeStore()` で dark/light 切替
- `makeTokens(isDark)` → `ThemeTokens` オブジェクト生成
- 全コンポーネントに `t: ThemeTokens` を props で渡す
- 色: `t.bg`, `t.surface`, `t.text`, `t.accent`, `t.red`, `t.blue` 等
- 文字サイズ: `useFontSizeStore()` → `getScaledFontSize(scale)` → `fs.sm`, `fs.lg` 等

## 7. 命名規則

| 対象 | 規則 | 例 |
|---|---|---|
| コンポーネントファイル | `PascalCase.tsx` | `PostBubble.tsx` |
| 非コンポーネントファイル | `camelCase.ts` | `scoring.ts` |
| コンポーネント | `export default function PascalCase()` | default export 必須 |
| hooks | `use` プレフィックス | `useAuth`, `useTheme` |
| 定数 | `UPPER_SNAKE_CASE` | `CAT_META`, `TIME_TABS` |
| 型 | `PascalCase` | `Post`, `Comment` |
| スタイル | ファイル末尾に定義 | `StyleSheet.create({})` |

## 8. コーディングルール（重要）

- **1ファイル500行以内を目標**
- **コメントは「なぜ」だけ書く**
- **NativeWind/Tailwind使用禁止** — StyleSheet + style prop統一
- **Supabase呼び出しは `lib/` に関数化** — 画面から直接書かない
- **Zustand はUI状態のみ** — サーバーデータは React Query
- **ネイティブモジュール原則禁止** — JS/TS のみで動作するライブラリを選ぶ
- **不要な import / 変数は即削除**
- **console.error は `if (__DEV__)` で本番抑制**

## 9. 既知の制約・注意点

| 項目 | 詳細 |
|---|---|
| **React Compiler** | Reanimated 4.1 と互換性なし。`babel.config.js` で `"react-compiler": false` に設定済み。開発モードでエラーが出るが本番では問題なし |
| **Google Maps 課金** | `LazyMapView` で遅延読み込み。Places Autocomplete のみ使用（Details API不使用）。APIキー制限未設定（リリース前に設定必要） |
| **Sentry** | 初期化コード作成済みだがDSN未設定 |
| **DEFAULT_RADIUS** | `constants/location.ts` で 50000m（50km）— 初期フェーズ用。ユーザー増加に応じて縮小予定 |
| **Reanimated** | 27ファイルで使用。ホーム画面周辺は RN 組み込み Animated に一部移行済み |
