# アーキテクチャ

> 更新日: 2026-03-28

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| フレームワーク | Expo ~54 + React Native 0.81 + TypeScript 5.9 |
| ルーティング | Expo Router（ファイルベース） |
| バックエンド | Supabase（Auth / Postgres / PostGIS / Storage / Realtime） |
| データ取得 | React Query（`@tanstack/react-query`） |
| フォーム | React Hook Form + Zod |
| 状態管理 | Zustand（UIのみ）、React Query（サーバーデータ） |
| アニメーション | React Native Reanimated ~4.1.1 |
| 位置情報 | expo-location（GPS + reverseGeocode） |
| 地図・場所検索 | react-native-maps + Google Places API（遅延読み込み） |

## データフロー

```
画面 (app/) → hooks (useXxx) → lib (fetchXxx / createXxx) → Supabase
```

- 画面から Supabase を直接呼ばない
- `lib/` にクエリ関数、`hooks/` に React Query ラッパー
- 作成・更新・削除は `useMutation` → 成功後に `invalidateQueries`

## 状態管理の使い分け

| データ種別 | 管理方法 |
|-----------|---------|
| サーバーデータ | React Query |
| テーマ・フォントサイズ | Zustand（`themeStore`, `fontSizeStore`） |
| タブバッジ | Zustand（`badgeStore`） |
| ゲストガード表示 | Zustand（`guestSheetStore`） |
| トースト通知 | Zustand（`toastStore`） |
| 認証状態 | Context Provider（`useAuth`） |
| 位置情報 | `useLocation` フック |

## 場所指定の3段階設計

| レベル | 操作 | API | コスト |
|--------|------|-----|--------|
| Lv1 | 現在地で投稿（デフォルト） | expo-location reverseGeocode | 無料 |
| Lv2 | 場所を検索 | Google Places Autocomplete | $2.83/1000 |
| Lv3 | 地図で見る（閲覧時） | Google Maps SDK | $7/1000 |
