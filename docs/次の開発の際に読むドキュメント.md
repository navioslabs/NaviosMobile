## 2. 開発環境の起動手順

### 2.1 エミュレーター起動

1. **Android Studio** を開く
2. **Device Manager**（右上のスマホアイコン）から仮想デバイスを起動
3. エミュレーターが完全に起動するまで待つ（ホーム画面が表示されるまで）

### 2.2 ビルド（初回 or ネイティブモジュール追加後）

```bash
npx expo run:android
```

- 初回は数分かかる
- ネイティブモジュール（`expo-location`, `expo-image-picker` 等）を追加した後にも必要
- ビルドが成功すると、エミュレーターにアプリがインストールされる

### 2.3 Expo Go での開発（QRコードモード）

ビルド済みの場合、通常の開発は Expo Go が速い:

```bash
npx expo start -c
```

- `-c` はキャッシュクリア（問題が起きたら必ずつける）
- ターミナルに QR コードが表示される
- **Android**: Expo Go アプリで QR コードを読み取る
- **iOS**: カメラアプリで QR コードを読み取る → Expo Go が開く

### 2.4 実機開発

1. スマホとPCを **同じWi-Fi** に接続
2. `npx expo start` を実行
3. QR コードをスマホで読み取る
4. Expo Go アプリ内でリアルタイムに反映される

**注意**: USB接続は不要。Wi-Fiだけでいい。

### 2.5 開発モードの切替

| コマンド | 用途 |
|---------|------|
| `npx expo start` | 開発サーバー起動 |
| `npx expo start -c` | キャッシュクリア + 起動 |
| `npx expo run:android` | Androidネイティブビルド |
| `npx expo run:ios` | iOSネイティブビルド（Mac必須） |
| `npm run web` | Web版で確認 |

## 3. エミュレータ強制終了

エミュレーターが閉じない・フリーズした場合:

### 関連プロセスの確認

```bash
tasklist | findstr /i "qemu emulator adb"
```

対象プロセス:

| プロセス名 | 役割 |
|-----------|------|
| `qemu-system-x86_64.exe` | エミュレータ本体（最もメモリ消費大） |
| `emulator.exe` | エミュレータランチャー |
| `adb.exe` | Android Debug Bridge |

### 全プロセスを強制終了

```bash
taskkill /F /IM qemu-system-x86_64.exe
taskkill /F /IM emulator.exe
adb kill-server
```

**注意**: `adb.exe` は `taskkill /F` で殺してもフォークサーバーが自動再起動する。必ず `adb kill-server` を使うこと。

### 再発防止

- **Windowsスタートアップ**: タスクマネージャー > スタートアップでエミュレータ関連を無効化
- **Android Studio設定**: `File > Settings > Tools > Emulator` で「Launch in a tool window」を確認
- **AVD設定**: 不要な仮想デバイスを削除

## 5. 知っておくべきアーキテクチャ

### データの流れ

```
画面 (app/) → hooks (useXxx) → lib (fetchXxx / createXxx) → Supabase
```

- 画面から Supabase を直接呼ばない
- `lib/` にクエリ関数、`hooks/` に React Query ラッパー
- 作成・更新・削除は `useMutation` → 成功後に `invalidateQueries`

### 状態管理の使い分け

| データ種別 | 管理方法 |
|-----------|---------|
| サーバーデータ（投稿、プロフィール等） | React Query |
| UI状態（テーマ、フォントサイズ） | Zustand |
| 認証状態 | Context Provider（`useAuth`） |
| 位置情報 | `useLocation` フック |

### スコアリング

| スコア | 用途 | 計算場所 |
|--------|------|---------|
| Pulse スコア（0〜100） | ちかく画面ヘッダーの「盛り上がり度」 | クライアント |
| マッチスコア（0〜100） | 投稿カードの関連度バッジ | クライアント |

詳細は `docs/scoring-guide/スコアリング設計.md` を参照。

### 位置情報の更新

- ちかく画面: `watchPositionAsync`（50m間隔、15秒最低間隔）
- 100m移動で `invalidateQueries` → サーバーに再問い合わせ
- `refetchInterval: 60秒` で立ち止まっていても鮮度維持
- タブ非アクティブ時は監視停止（バッテリー保護）

新しいマイグレーションSQLを作った場合:
1. Supabase Dashboard → **SQL Editor**
2. **New query** → SQLをペースト → **Run**

マイグレーションファイルは `supabase/migrations/` に番号付きで管理。

## 8. よくある問題と対処

| 問題 | 対処 |
|------|------|
| ホットリロードが反映されない | `npx expo start -c` でキャッシュクリア再起動 |
| `crypto` エラー | `react-native-get-random-values` が import されているか確認 |
| 画像アップロードエラー | Supabase Storage のバケットRLSを確認 |
| 位置情報が取れない | エミュレーター: Extended Controls > Location で座標を設定 |
| TypeScriptエラー確認 | `npx tsc --noEmit` を実行 |
| ビルドエラー（ネイティブ） | `npx expo run:android` で再ビルド |
| エミュレーターが閉じない | 上記「エミュレータ強制終了」参照 |
