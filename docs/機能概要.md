# NaviOs 機能概要書

> 最終更新: 2026-03-28
> ステータス: Supabase 接続済み + UI/UX改善フェーズ完了
> 用途: 現状UIとMVP仕様の共通理解をまとめる

---

## 1. 概要

NaviOs は、地域の投稿・会話・近接情報を扱うモバイルアプリである。

現在の重点は以下の4つ。

- フィードで地域投稿を見つける
- トークで近所の声を確認する
- さがすで条件に合う情報を見つける
- ちかくで距離ベースの情報を確認する

本書では、`現状実装に近い仕様` と `MVPで扱う機能` を優先して整理する。

## 4. フィード

### 4.1 役割

- 地域投稿を日付・カテゴリ・優先軸で確認する主画面

### 4.2 現在の表示要素

- 日付ピッカー
- サマリーカード
- カテゴリチップ
- 投稿カード一覧
- 空状態表示

### 4.3 並び順とフィルタ

| フィルタ                | 条件                         |
| ----------------------- | ---------------------------- |
| デフォルト              | 選択日付に該当する投稿を表示 |
| 注目 (`top`)            | `matchScore` 降順            |
| 近く (`nearby`)         | `distance <= 200` で絞り込み |
| まもなく終了 (`urgent`) | `timeLeft` 昇順              |

補足:

- モックでは「今日 / 明日 / それ以降」に近い日付切替
- 先頭カードは `featured` 表示される
- 実データ接続後は `created_at` と `expires_at` の扱いを仕様化する必要がある

### 4.4 投稿カードで扱う情報

- 投稿者名
- アバター
- 認証済みフラグ
- 画像
- キャプション
- カテゴリ
- 距離
- `matchScore`
- 残り時間
- いいね数
- 混雑度

- 条件に合う投稿を検索する画面

### 6.2 現在の表示要素

### 6.3 検索仕様

- キャプションに対する部分一致

- 最大6件、`likes_count` 降順

### 6.6 今後の整理ポイント

## 7. ちかく

### 7.1 役割

- 位置ベースで近隣投稿を確認する画面
- 投稿閲覧よりも「今どこへ行くか」の判断に寄せる

### 7.2 現在の表示要素

- `周辺をスキャン中` ヘッダー
- 盛り上がり度表示
- 更新時刻表示
- 距離帯セクション
- 近隣投稿一覧
- 空セクション時の案内

### 7.3 距離帯

| セクション | 距離範囲  | 色  |
| ---------- | --------- | --- |
| すぐ近く   | 0〜200m   | 緑  |
| 徒歩圏内   | 200〜500m | 橙  |
| その他     | 500m超    | 灰  |

### 7.4 盛り上がり度とスキャン表示

`ちかく` のヘッダーでは、`周辺をスキャン中` と `盛り上がり度` を合わせて表示する。

現在の意図:

- `周辺をスキャン中`: 周囲の情報を見ている画面であることを伝える
- `盛り上がり度`: 周辺エリアの活気を数値で補足する
- `LIVE / 更新時刻`: リアルタイム感の補助

今後の改善ポイント:

- 盛り上がり度の算出根拠を仕様として定義する
- `LIVE` の意味を「現在地近辺の情報を表示中」として明文化する
- 実データ接続後は更新時刻と取得元を連動させる

補足:

- 現状では `盛り上がり度` は `ちかく` ヘッダー内に統合されている
- そのため、旧仕様の `AIタブ上部 Pulseスコアカード` 前提は今後の文書では使わない

## 8. Match Score / 盛り上がり度

### 8.1 投稿単位の `matchScore`

各投稿は `0〜100` の `matchScore` を持つ。

用途:

- 注目度の比較
- おすすめ表示の優先度
- ちかく / さがす での見つけやすさ向上

### 8.2 表示ルール

| スコア範囲 | 色  | 補足   |
| ---------- | --- | ------ |
| 80〜100    | 緑  | 高注目 |
| 60〜79     | 橙  | 中注目 |
| 0〜59      | 灰  | 通常   |

補足:

- `85以上` の投稿は `HOT` 表示候補
- `featured` 表示の優先条件の1つとして使う

### 8.3 エリア単位の盛り上がり度

`ちかく` ヘッダーに表示される `82 / 100` のような値は、投稿単位の `matchScore` とは別に、周辺エリア全体の活気指標として扱う。

今後必要な設計:

- 何件の投稿を対象にするか
- 距離と時間をどう重みづけするか
- 混雑度や期限切れ投稿をどう扱うか

## 9. 投稿機能

### 9.1 フィード投稿

作成画面: `/post`

入力項目:

- カテゴリ
- タイトル
- 詳細説明
- 写真
- 位置情報

現状整理:

- タイトルは必須
- カテゴリはUI上で先に選ばれる

- 保存処理は未接続

### 9.2 トーク投稿

作成画面: `/talk-post`

入力項目:

- メッセージ本文
- 写真
- 位置情報ラベル

現状整理:

- 本文は実質必須
- 140文字制限あり
- 位置情報ラベルは自動付与想定
- 保存処理は未接続

## 12. 位置情報

現状:

- 越谷市固定のモック
- 距離表示あり
- 距離帯フィルタあり
- 地図表示なし

仕様メモ:

- 徒歩時間は `距離 ÷ 80m/分` を基準に算出可能
- 投稿への位置ラベル付与を想定
- 実機実装時は位置情報拒否時の代替表示が必要

- フィード一覧表示

- ちかくの距離帯表示
- 投稿作成UIと保存
- 基本プロフィール
- テーマ / 文字サイズ

### MVP後でよいもの

- 音声検索
- シェア
- フォロー
- 実績判定ロジック高度化
- 地図表示
- 高度な通知
