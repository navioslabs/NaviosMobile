# AI実装プロンプト・必要書類・今後タスク

> 対象: NaviOs
> 更新日: 2026-03-25
> 用途: UI改善後の実装フェーズで、AIに読み込ませるプロンプト、作成すべき書類、今後のタスクを整理する

---

## 1. 現在地

現状は以下の前提で進める。

- UIモックはかなり具体化されている
- 今後は「見た目」よりも「実装設計」「データ設計」「API設計」「運用設計」の比重が上がる
- 技術方針:
  - Expo ~55
  - React Native 0.83
  - TypeScript 5.9
  - Supabase
  - React Query
  - React Hook Form + Zod
- ネイティブモジュールは可能な限り避ける

この段階では、AIに対して「UIを考えて」よりも、

- 要件を整理する
- 実装単位に分解する
- DB / API / 状態管理を設計する
- タスクに落とす

ためのプロンプトが重要になる。

---

## 2. AIに読み込ませる基本コンテキスト

以下は、今後どのプロンプトでも先頭に付ける共通文として使う。

```text
あなたは Expo / React Native / TypeScript / Supabase / React Query に強いシニアエンジニア兼テックリードです。

対象プロジェクト:
- 名前: NaviOs
- 種別: 地域情報共有モバイルアプリ
- 現在段階: UIモックは概ね固まっており、これから実装設計と開発を進める段階
- 技術スタック: Expo ~55, React Native 0.83, TypeScript 5.9, Supabase, React Query, React Hook Form, Zod
- 状態管理方針: サーバーデータは React Query、UI状態はローカル state または Zustand
- 言語: 日本語のみ
- 制約: ネイティブモジュールは可能な限り避ける

前提:
- 画面UIだけでなく、実装しやすさ、保守性、コスト、将来拡張も重視してください
- Supabase 呼び出しは画面から直接書かず、lib層へ分離する前提で考えてください
- 抽象論ではなく、実装に落とせる粒度で提案してください
```

## 3. AI用プロンプト集

### 3.1 要件整理プロンプト

用途:
- 画面単位のふわっとしたアイデアを、実装可能な要件へ落とす

```text
[共通コンテキストを貼る]

以下の機能について、実装要件を整理してください。

対象機能:
[ここに機能名を書く]

欲しい出力:
1. 機能の目的
2. ユーザー価値
3. 画面上の振る舞い
4. 必要なデータ
5. バリデーション
6. エラーパターン
7. 権限制御の要否
8. MVPで必要な範囲
9. 後回しでよい範囲

条件:
- 実装に落とせる粒度で整理してください
- 曖昧な部分は合理的な仮定を置いてください
- Supabase と React Query を使う前提で考えてください
```

### 3.2 画面仕様書作成プロンプト

- 各画面の仕様書たたき台を作る

```text

[例: ホーム / さがす / ちかく / ひとこと / 設定 / 投稿]

出力形式:

5. 表示データ
6. ローディング時の表示

8. エラー状態
9. 画面遷移
10. API / Query / Mutation の候補
11. 実装上の注意点

要件:
- React Native で実装できる粒度で書く
- UI仕様だけでなく、状態遷移も含める
```

## 4. これから必要になる書類

優先度順で並べる。

### 優先度 High

1. 要件定義書
   - MVPで何を作るかを固定する
2. 画面仕様書
   - 各画面の状態、操作、表示ルールを定義する
3. データ設計書
   - Supabase テーブル、RLS、Storage 方針を整理する
4. API / lib設計書
   - Supabase 呼び出し関数、React Query 設計を整理する
5. タスク分解書
   - 実装順と依存関係を管理する

### 優先度 Middle

6. 状態管理方針書
   - React Query と Zustand / local state の責務分離
7. バリデーション設計書
   - React Hook Form + Zod のルール定義
8. エラーハンドリング設計書
   - 画面表示メッセージと例外方針
9. 認証 / 権限制御設計書
   - Auth, RLS, ゲスト権限の扱い
10. テスト観点一覧
   - 重要ケースの洗い出し

### 優先度 Low

11. 運用設計書
   - 障害時対応、ログ確認、問い合わせ導線
12. リリースチェックリスト
13. KPI / 計測設計書
14. 将来機能バックログ
