## 1. ホーム（フィード）

**ルート**: `/(tabs)/`
**目的**: 地域の投稿一覧を日付・カテゴリ・並び順で閲覧する

### UI構成

| コンポーネント | 役割 |
|---------------|------|
| `Header` | アプリヘッダー |
| `DatePicker` | 日付タブ（今日〜7日後） |
| `FeedSummary` | フィルタ切替（トップ / 近い / 急ぎ） |
| `CategoryChips` | カテゴリフィルタ（全て / 物資 / イベント / 近助 / 行政） |
| `FeedPostCard` | 投稿カード（先頭はfeatured表示） |

### ユーザー操作

| 操作 | 動作 |
|------|------|
| 日付タブ切替 | `hoursAgo` でフィルタ |
| カテゴリタップ | カテゴリフィルタ |
| フィルタ切替 | `matchScore` / `distance` / `timeLeft` でソート |
| カードタップ | `/feed/[id]` へ遷移 |
| プルダウン | リフレッシュ |
| FABタップ | 投稿作成画面へ |

### データ要件

| 項目 | 現在 | 本番 |
|------|------|------|
| データソース | `FEED_POSTS`（モック10件） | `usePosts(filters)` → Supabase |
| フィルタ | クライアント側フィルタ | サーバーサイドクエリ |

### 状態表示

| 状態 | 表示 |
|------|------|
| ローディング | `StateView type="loading"` |
| 空 | 「この日の投稿はまだありません」+ 「今日を見る」「投稿する」ボタン |
| エラー | `StateView type="error"` + リトライ |

## 2. ちかく

**ルート**: `/(tabs)/nearby`
**目的**: 現在地から近い順に投稿を距離区分で表示する

### UI構成

| コンポーネント | 役割 |
|---------------|------|
| `Header` | アプリヘッダー |
| `ScanHeader` | 「周辺をスキャン中」+ 件数表示 |
| `DistanceSectionHeader` | 距離区分ヘッダー |
| `NearbyPostItem` | 投稿アイテム（先頭はfeatured） |

### 距離区分

| 区分 | 範囲 | 色 |
|------|------|-----|
| すぐ近く | 0〜200m | 緑 `#00D4A1` |
| 徒歩圏内 | 200〜500m | 橙 `#F5A623` |
| その他 | 500m〜 | グレー |

### データ要件

| 項目 | 現在 | 本番 |
|------|------|------|
| データソース | `NEARBY_POSTS`（モック13件） | `useNearbyPosts(coords)` → PostGIS |
| 位置情報 | モック距離値 | `expo-location` で取得した現在地座標 |
| 距離計算 | モック値 | `ST_Distance` でサーバー側算出 |

## 4. さがす

**ルート**: `/(tabs)/ai`
**目的**: テキスト検索で投稿を探す + AIレコメンド

### UI構成

| コンポーネント | 役割 |
|---------------|------|
| ロゴ | 「Navios AI」 |
| 検索バー | テキスト入力 + クリア/マイク |
| `SuggestionChips` | 検索候補チップ |
| `PulseEventCard` | 「今すぐ行くべき順」カード |
| `PulseScoreCard` | 検索結果カード |

### モード

| モード | 条件 | 表示 |
|--------|------|------|
| 通常 | 検索クエリなし | SuggestionChips + PulseEventCard一覧 |
| 検索中 | ローディング中 | スピナー |
| 検索結果あり | 結果 > 0 | 「{query}の検索結果 — {count}件」+ カード一覧 |
| 検索結果なし | 結果 = 0 | 空状態 + 別のSuggestionChips |

### 検索ロジック（現在）

```
caption.includes(query) || user.name.includes(query) || category.includes(query)
```

**本番**: Supabase の `ilike` or `textSearch` に移行

## 6. 投稿作成

**ルート**: `/post`（モーダル）
**目的**: フィード投稿を新規作成する

### フォームフィールド

| フィールド | 必須 | UI | 現在の実装 |
|-----------|------|-----|-----------|
| カテゴリ | はい | 4ボタン選択 | `useState` |
| タイトル | はい | TextInput | `useState` |
| 詳細 | いいえ | TextInput (multiline) | `useState` |
| 写真 | いいえ | カメラ / 画像選択 | 未実装（UI枠のみ） |
| 場所 | いいえ | 場所選択ボタン | 未実装（UI枠のみ） |

### 送信条件

- カテゴリ選択済み かつ タイトルが空でない → 送信ボタン有効

### 本番移行

- フォーム → React Hook Form + Zod
- 送信 → `useCreatePost()` mutation
- 画像 → `uploadImage()` → URL を `image_url` に保存
- 位置 → `expo-location` で座標取得

## 8. 投稿詳細

**ルート**: `/feed/[id]`
**目的**: 投稿の詳細情報を表示する

### UI構成

| エリア | 内容 |
|--------|------|
| ヒーロー画像 | 260px、グラデーションオーバーレイ |
| カテゴリ | ピル + 「急ぎ」バッジ（timeLeft ≤ 60分） |
| タイトル + 距離 | 大文字タイトル + 距離バッジ |
| 発信者情報 | アバター、名前、認証バッジ、時刻 |
| 概要 | 本文テキスト |
| 期限カード | 「本日 HH:MM まで」等 |
| 必要な持ち物 | カテゴリ別（admin: 身分証等、event: 飲み物等、help: 軍手等） |
| アクション | 共有、保存、通報 |

### 通報モーダル

- `ReportModal` で6種類の理由から選択
- `targetType: "feed"`, `targetId: post.id`

## 11. プロフィール編集

**ルート**: `/profile/edit`
**目的**: 自分のプロフィール情報を編集する

### フォームフィールド

| フィールド | 必須 | 制限 | 現在の実装 |
|-----------|------|------|-----------|
| アバター画像 | いいえ | 画像選択 | UI枠のみ |
| 表示名 | はい | 最大20文字 | `useState` |
| 自己紹介 | いいえ | 最大150文字 | `useState` |
| 地域 | いいえ | テキスト | `useState` |
| プロフィール公開 | — | トグル | `useState` (初期: true) |
| 位置情報を表示 | — | トグル | `useState` (初期: true) |
| チェックイン履歴を公開 | — | トグル | `useState` (初期: false) |

### 保存条件

- 表示名が空でない → 保存ボタン有効（アクセントカラー）

### 本番移行

- フォーム → React Hook Form + Zod
- 保存 → `useUpdateProfile()` mutation
- アバター → `uploadImage('avatars', uri)` → URL を `avatar_url` に保存
