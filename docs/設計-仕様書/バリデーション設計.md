> 対象: NaviOs
> 更新日: 2026-03-25
> ステータス: 設計案（未実装 — React Hook Form + Zod はインストール済みだが未使用）

---

## 1. 方針

- バリデーションは **Zod スキーマ** で一元定義する
- フォームは **React Hook Form** の `useForm` + `zodResolver` で接続する
- エラーメッセージは **日本語** で統一する
- Zod スキーマは `lib/validations.ts` にまとめて定義する

```
lib/validations.ts  ← Zod スキーマ定義
  ↓
hooks/useXxxForm.ts ← useForm + zodResolver でフォーム管理
  ↓
app/xxx.tsx         ← 画面で使用
```

---

## 2. 共通バリデーションルール

| ルール | Zod | メッセージ |
|--------|-----|-----------|
| 必須 | `z.string().min(1)` | 「入力してください」 |
| メール | `z.string().email()` | 「有効なメールアドレスを入力してください」 |
| 最小文字数 | `z.string().min(n)` | 「{n}文字以上で入力してください」 |
| 最大文字数 | `z.string().max(n)` | 「{n}文字以内で入力してください」 |
| パスワード確認 | `.refine()` | 「パスワードが一致しません」 |

## 3. フォーム別 Zod スキーマ

### 3.1 ログイン（loginSchema）

```typescript
const loginSchema = z.object({
  email: z.string()
    .min(1, '入力してください')
    .email('有効なメールアドレスを入力してください'),
  password: z.string()
    .min(1, '入力してください'),
});
```

| フィールド | 型 | 必須 | ルール |
|-----------|-----|------|--------|
| email | string | はい | メール形式 |
| password | string | はい | 空でない |

---

### 3.2 サインアップ（signupSchema）

```typescript
const signupSchema = z.object({
  displayName: z.string()
    .min(1, '入力してください')
    .max(20, '20文字以内で入力してください'),
  email: z.string()
    .min(1, '入力してください')
    .email('有効なメールアドレスを入力してください'),
  password: z.string()
    .min(8, '8文字以上で入力してください'),
  confirmPassword: z.string()
    .min(1, '入力してください'),
}).refine(
  (data) => data.password === data.confirmPassword,
  { message: 'パスワードが一致しません', path: ['confirmPassword'] }
);
```

| フィールド | 型 | 必須 | ルール |
|-----------|-----|------|--------|
| displayName | string | はい | 1〜20文字 |
| email | string | はい | メール形式 |
| password | string | はい | 8文字以上 |
| confirmPassword | string | はい | password と一致 |

### 3.3 投稿作成（createPostSchema）

```typescript
const createPostSchema = z.object({
  category: z.enum(['stock', 'event', 'help', 'admin'], {
    required_error: 'カテゴリを選択してください',
  }),
  title: z.string()
    .min(1, 'タイトルを入力してください')
    .max(100, '100文字以内で入力してください'),
  content: z.string()
    .max(2000, '2000文字以内で入力してください')
    .optional()
    .or(z.literal('')),
  imageUri: z.string().optional(),
  locationText: z.string().optional(),
});
```

| フィールド | 型 | 必須 | ルール |
|-----------|-----|------|--------|
| category | enum | はい | stock / event / help / admin |
| title | string | はい | 1〜100文字 |
| content | string | いいえ | 最大2000文字 |
| imageUri | string | いいえ | — |
| locationText | string | いいえ | — |

## 4. React Hook Form 接続パターン

```typescript
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { createPostSchema } from '@/lib/validations';

type CreatePostForm = z.infer<typeof createPostSchema>;

const { control, handleSubmit, formState: { errors, isValid } } = useForm<CreatePostForm>({
  resolver: zodResolver(createPostSchema),
  defaultValues: { category: 'stock', title: '', content: '' },
  mode: 'onChange',  // リアルタイムバリデーション
});
```

## 5. エラー表示方針

| 場所 | 表示方法 |
|------|---------|
| フィールド下 | 赤テキストでエラーメッセージ |
| 文字数カウンター | 制限超過時に赤色に変化（既存UI踏襲） |
| 送信ボタン | `isValid` が false の間はグレー表示 + disabled |
| サーバーエラー | フォーム上部にアラートバナー（認証失敗、ネットワークエラー等） |

---

## 6. 移行の優先順位

| 順位 | フォーム | 理由 |
|------|---------|------|
| 1 | ログイン / サインアップ | 認証実装時に同時対応 |
| 2 | 投稿作成 | 最も使用頻度が高い |
| 3 | ひとこと投稿 | シンプルで移行しやすい |
| 4 | プロフィール編集 | フィールド数が多く効果が高い |
| 5 | 通報 | ReportModal 内、優先度低め |
| 6 | コメント / 返信 | 単一フィールドで簡素 |
