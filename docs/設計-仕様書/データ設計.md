## 1. テーブル一覧

| テーブル名     | 用途                                    | 依存                |
| -------------- | --------------------------------------- | ------------------- |
| `profiles`     | ユーザープロフィール                    | `auth.users`        |

## 2. テーブル詳細

### 2.1 profiles

Supabase Auth の `auth.users` と 1:1 で紐づくプロフィールテーブル。

| カラム          | 型            | 制約                     | 説明                       |
| --------------- | ------------- | ------------------------ | -------------------------- |
| `id`            | `uuid`        | PK, FK → `auth.users.id` | ユーザーID                 |
| `display_name`  | `text`        | NOT NULL, max 20         | 表示名                     |
| `avatar_url`    | `text`        | NULL可                   | アバター画像URL（Storage） |
| `bio`           | `text`        | NULL可, max 150          | 自己紹介                   |
| `location_text` | `text`        | NULL可                   | 地域テキスト（例: 越谷市） |
| `is_verified`   | `boolean`     | DEFAULT false            | 認証済みバッジ             |
| `is_public`     | `boolean`     | DEFAULT true             | プロフィール公開設定       |
| `show_location` | `boolean`     | DEFAULT true             | 位置情報表示設定           |
| `show_checkins` | `boolean`     | DEFAULT false            | チェックイン履歴公開設定   |
| `created_at`    | `timestamptz` | DEFAULT now()            | 作成日時                   |
| `updated_at`    | `timestamptz` | DEFAULT now()            | 更新日時                   |

**index**: `display_name` (検索用)

### 2.2 posts

フィード投稿。カテゴリ別の地域情報。

| カラム           | 型                      | 制約                          | 説明                                     |
| ---------------- | ----------------------- | ----------------------------- | ---------------------------------------- |
| `id`             | `uuid`                  | PK, DEFAULT gen_random_uuid() | 投稿ID                                   |
| `author_id`      | `uuid`                  | FK → `profiles.id`, NOT NULL  | 投稿者                                   |
| `category`       | `text`                  | NOT NULL, CHECK               | `stock` / `event` / `help` / `admin`     |
| `title`          | `text`                  | NOT NULL                      | タイトル                                 |
| `content`        | `text`                  | NULL可                        | 詳細テキスト                             |
| `image_url`      | `text`                  | NULL可                        | 画像URL（Storage）                       |
| `location`       | `geography(Point,4326)` | NULL可                        | PostGIS座標                              |
| `location_text`  | `text`                  | NULL可                        | 場所名テキスト                           |
| `deadline`       | `timestamptz`           | NULL可                        | 期限（在庫の売り切れ時刻等）             |
| `crowd`          | `text`                  | NULL可                        | 混雑度: `crowded` / `moderate` / `empty` |
| `is_featured`    | `boolean`               | DEFAULT false                 | 注目投稿フラグ                           |
| `likes_count`    | `integer`               | DEFAULT 0                     | いいね数（非正規化）                     |
| `comments_count` | `integer`               | DEFAULT 0                     | コメント数（非正規化）                   |
| `created_at`     | `timestamptz`           | DEFAULT now()                 | 投稿日時                                 |
| `updated_at`     | `timestamptz`           | DEFAULT now()                 | 更新日時                                 |

**index**:

- `author_id`
- `category`
- `created_at DESC`
- `location` (GIST, PostGIS空間インデックス)
- `deadline` (期限切れフィルタ用)

**計算フィールド（クエリ時に算出）**:

- `distance`: クライアントの現在地との距離（PostGIS `ST_Distance`）
- `match_score`: カテゴリ・距離・新しさから算出（Postgres関数 or クライアント側）
- `time_left`: `deadline - now()` から算出

### 2.6 likes

投稿・ひとこと・コメント・返信すべてのいいねを1テーブルで管理。

| カラム        | 型            | 制約                          | 説明                                  |
| ------------- | ------------- | ----------------------------- | ------------------------------------- |
| `id`          | `uuid`        | PK, DEFAULT gen_random_uuid() | いいねID                              |
| `user_id`     | `uuid`        | FK → `profiles.id`, NOT NULL  | いいねしたユーザー                    |
| `target_type` | `text`        | NOT NULL, CHECK               | `post` / `talk` / `comment` / `reply` |
| `target_id`   | `uuid`        | NOT NULL                      | 対象のID                              |
| `created_at`  | `timestamptz` | DEFAULT now()                 | いいね日時                            |

**UNIQUE**: `(user_id, target_type, target_id)` — 同一ユーザーの重複いいね防止

**index**: `(target_type, target_id)` — いいね数カウント用

| `reason`      | `text`        | NOT NULL, CHECK               | 通報理由（下記参照）       |

**通報理由 enum**:

- `spam` — スパム・宣伝
- `inappropriate` — 不適切な内容

## 3. RLS（Row Level Security）方針

すべてのテーブルで RLS を有効化する。

| テーブル       | SELECT                             | INSERT                  | UPDATE             | DELETE                 |
| -------------- | ---------------------------------- | ----------------------- | ------------------ | ---------------------- |
| `profiles`     | 全員（`is_public = true` or 本人） | Auth trigger で自動作成 | 本人のみ           | 不可                   |
| `posts`        | 全員                               | ログインユーザー        | 投稿者本人のみ     | 投稿者本人のみ         |
| `talks`        | 全員                               | ログインユーザー        | 投稿者本人のみ     | 投稿者本人のみ         |
| `comments`     | 全員                               | ログインユーザー        | コメント者本人のみ | コメント者本人のみ     |
| `talk_replies` | 全員                               | ログインユーザー        | 返信者本人のみ     | 返信者本人のみ         |
| `likes`        | 全員                               | ログインユーザー        | 不可               | 本人のみ（いいね取消） |
| `reports`      | 本人のみ                           | ログインユーザー        | 不可               | 不可                   |

**ゲスト（未ログイン）**: SELECT のみ許可。投稿・いいね・コメントにはログイン必須。

## 5. PostGIS

**有効化が必要な拡張**:

```sql
CREATE EXTENSION IF NOT EXISTS postgis;
```

**使用箇所**:

- `posts.location` — 投稿の座標
- `talks.location` — ひとことの座標

**距離計算クエリ例**:

```sql
SELECT *,
  ST_Distance(
    location,
    ST_SetSRID(ST_MakePoint(:lng, :lat), 4326)::geography
  ) AS distance_m
FROM posts
WHERE ST_DWithin(
  location,
  ST_SetSRID(ST_MakePoint(:lng, :lat), 4326)::geography,
  5000  -- 5km以内
)
ORDER BY distance_m ASC;
```

**ちかく画面の距離区分**:

- すぐ近く: 0〜200m
- 徒歩圏内: 200〜500m
- その他: 500m〜

## 6. 非正規化フィールドの更新方針

`likes_count`, `comments_count`, `replies_count` は非正規化カウンター。

**更新方法**: Postgres トリガー関数で自動更新する。

```sql
-- いいね追加時に対象テーブルのカウントを+1
-- いいね削除時に-1
-- コメント・返信も同様
```

トリガーにすることで、アプリ側でカウント管理する必要がなくなる。
