# NaviOs スコアリング設計

> 更新日: 2026-03-25
> 対象: ちかく画面のPulseスコア、投稿のマッチスコア

---

## 1. 概要

NaviOs には2種類のスコアリングがある。

| スコア           | 用途                                             | 計算場所     | 範囲   |
| ---------------- | ------------------------------------------------ | ------------ | ------ |
| **Pulse スコア** | ちかく画面のヘッダーに表示。エリアの盛り上がり度 | クライアント | 0〜100 |
| **マッチスコア** | 個別投稿カードに表示。ユーザーとの関連度         | クライアント | 0〜100 |

---

## 2. Pulse スコア（盛り上がり度）

### 表示場所

`components/features/nearby/ScanHeader.tsx` の「盛り上がり度 xx / 100」

### 算出ロジック

```
Pulse = 投稿数スコア(40pt) + 近接率スコア(30pt) + 新着率スコア(30pt)
```

| 要素   | 配点      | 計算式                          | 意味                                             |
| ------ | --------- | ------------------------------- | ------------------------------------------------ |
| 投稿数 | 最大 40pt | `min(投稿数, 20) / 20 × 40`     | 周辺の投稿が多いほど盛り上がっている。20件で上限 |
| 近接率 | 最大 30pt | `200m以内の件数 / 全件数 × 30`  | 近い投稿の割合が高いほどスコアが高い             |
| 新着率 | 最大 30pt | `直近1時間の件数 / 全件数 × 30` | 新しい投稿が多いほど活発                         |

### 具体例

| 状況         | 投稿数 | 200m以内 | 1時間以内 | スコア                |
| ------------ | ------ | -------- | --------- | --------------------- |
| 活発なエリア | 15件   | 8件      | 6件       | 30 + 16 + 12 = **58** |
| 駅前の朝市   | 20件   | 15件     | 12件      | 40 + 23 + 18 = **81** |
| 閑散なエリア | 3件    | 1件      | 0件       | 6 + 10 + 0 = **16**   |
| 投稿なし     | 0件    | —        | —         | **0**                 |

### HOT バッジ

- スコアが **60以上** のとき、「HOT」バッジが表示される
- HOT バッジは脈動アニメーション付き（1.4秒周期）

### 実装ファイル

```typescript
// components/features/nearby/ScanHeader.tsx
function calcPulseScore(
  postCount: number,
  closeCount: number,
  recentCount: number,
): number {
  if (postCount === 0) return 0;
  return Math.min(
    100,
    Math.round(
      (Math.min(postCount, 20) / 20) * 40 +
        (closeCount / Math.max(postCount, 1)) * 30 +
        (recentCount / Math.max(postCount, 1)) * 30,
    ),
  );
}
```

### 入力データの取得元

| 入力          | 取得元                                 |
| ------------- | -------------------------------------- |
| `postCount`   | `useNearbyPosts` の結果件数            |
| `closeCount`  | `distance_m <= 200` でフィルタした件数 |
| `recentCount` | `created_at` が直近1時間以内の件数     |

集計は `app/(tabs)/index.tsx` で行い、props として ScanHeader に渡す。

## 3. マッチスコア（投稿の関連度）

### 表示場所

`components/features/nearby/NearbyPostItem.tsx` の MatchBadge
`components/features/feed/FeedPostCard.tsx` の Featured 判定

### 算出ロジック

現在は**距離ベースの簡易スコア**のみ。

```
マッチスコア = max(0, 100 - floor(距離m / 50))
```

| 距離  | スコア | 解釈       |
| ----- | ------ | ---------- |
| 0m    | 100    | 最高マッチ |
| 50m   | 99     | ほぼ目の前 |
| 200m  | 96     | すぐ近く   |
| 500m  | 90     | 徒歩圏内   |
| 1000m | 80     | 少し歩く   |
| 2000m | 60     | やや遠い   |
| 5000m | 0      | 圏外       |

### 実装ファイル

```typescript
// lib/adapters.ts
export function calcMatchScore(distanceM: number): number {
  return Math.max(0, 100 - Math.floor(distanceM / 50));
}
```

### 将来の拡張案

現在は距離のみだが、将来的に以下の要素を加重できる。

| 要素         | 配点案 | 意味                       |
| ------------ | ------ | -------------------------- |
| 距離         | 50pt   | 近いほど高い               |
| カテゴリ嗜好 | 20pt   | ユーザーがよく見るカテゴリ |
| 鮮度         | 15pt   | 新しいほど高い             |
| 人気度       | 15pt   | いいね数・コメント数       |

拡張時は `lib/scoring.ts` を新設し、サーバー側（Postgres関数）でも同様のロジックを持たせることを推奨。

## 4. 距離の更新タイミング

### 位置情報の取得方式

| 画面         | 方式                                 | パラメータ            |
| ------------ | ------------------------------------ | --------------------- |
| ちかく画面   | `watchPositionAsync`（継続監視）     | 50m間隔、15秒最低間隔 |
| その他の画面 | `getCurrentPositionAsync`（1回取得） | —                     |

### サーバー再問い合わせの閾値

```
前回のクエリ座標からの移動距離 >= 100m → invalidateQueries
```

- 100m の理由: 200m セクションの半分。セクション境界をまたぐ前に必ず1回更新される
- 距離計算: クライアント側で Haversine 公式を使用（`lib/utils.ts`）

### 自動リフレッシュ

- `refetchInterval: 60000`（60秒ごと）— 立ち止まっていても鮮度を保つ
- `staleTime: 30000`（30秒）— 30秒以内の再取得はキャッシュを使う

### バッテリー保護

- タブ非アクティブ時は `watchPositionAsync` を停止（`useIsFocused` で制御）
- バックグラウンド監視は行わない

## 5. 距離計算の仕組み

### サーバー側（PostGIS）

`supabase/migrations/20260325000001_rpc_nearby.sql` の `get_nearby_posts` 関数が、ユーザー座標と投稿座標の距離をメートル単位で返す。

```sql
ST_Distance(
  p.location,
  ST_SetSRID(ST_MakePoint(user_lng, user_lat), 4326)::geography
) AS distance_m
```

### クライアント側（Haversine）

100m 閾値の判定に使用。サーバーへの問い合わせを間引くための軽量計算。

```typescript
// lib/utils.ts
export function haversineDistance(lat1, lng1, lat2, lng2): number;
```

## 6. セクション分け

| セクション | 距離範囲  | カラー            | 意味         |
| ---------- | --------- | ----------------- | ------------ |
| すぐ近く   | 0〜200m   | `#00D4A1` (green) | 歩いて1〜2分 |
| 徒歩圏内   | 200〜500m | `#F5A623` (amber) | 歩いて3〜6分 |
| その他     | 500m〜    | `#8887A0` (gray)  | 少し遠い     |

- 0件のセクションは非表示（ただし「すぐ近く」は常に表示）
- 各セクション内は距離昇順でソート
