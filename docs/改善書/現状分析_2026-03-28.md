## 3. 技術的負債・既知の問題

### 要対応

| 問題 | 影響 | 対処法 |
|------|------|--------|
| DBマイグレーション未適用 | ゴースト投稿・バッジ・街の記憶が動かない | SQL Editorで `20260328000000` を実行 |
| 全投稿を一括取得 | 投稿増加時にパフォーマンス劣化 | サーバーフィルタ強化 |

### iOS 固有の注意点（今日のバグから学んだ）

| 注意点 | 詳細 |
|--------|------|
| lucide アイコン名リネーム | `Edit3` → `Pencil` 等。`icons.js` で `Pencil as Edit3` としてエイリアス済み |
| Reanimated strict mode | shared value をレンダー中に書き込むと iOS でクラッシュ。必ず `useEffect` 内で |
| usePreventRemove + useRef | ref の `.current` 変更は再レンダーを起こさない → ガード解除されない。state を使う |
| Metro キャッシュ分離 | iOS/Android で別キャッシュ。`npx expo start -c` + `rm -rf node_modules/.cache` |
| Provider の外でフック呼出 | `QueryClientProvider` の外で React Query フックを呼ぶとクラッシュ |

## 4. ファイル構成サマリー

```
app/
  (tabs)/          ... 4タブ画面 + _layout（ホーム=ちかく+フィード統合）
  (auth)/          ... ログイン・サインアップ
  feed/[id].tsx    ... 投稿詳細
  talk-detail/[id] ... トーク詳細
  profile/[id]     ... プロフィール
  profile/edit     ... プロフィール編集
  post.tsx         ... 投稿作成（モーダル）
  talk-post.tsx    ... トーク作成（モーダル）
  street-history   ... 街の記憶

components/
  features/        ... 機能別（ai, badges, feed, history, nearby, settings, talk）
  layout/          ... Header, Fab
  ui/              ... 汎用UI（Toast, StateView, ReportModal, GuestLoginSheet 等）

hooks/             ... 15カスタムフック
lib/               ... 17サービス層ファイル
stores/            ... 5 Zustandストア
constants/         ... 5定数ファイル
types/             ... 型定義
supabase/          ... 7マイグレーション
```
