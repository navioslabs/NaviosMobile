# NaviOs アニメーション実装提案書

> 対象: NaviOs 現行ビルド
> 更新日: 2026-03-25
> 位置づけ: UI演出フェーズの実装ガイド

---

## 1. 方針

NaviOs は「地域情報をすばやく見る」アプリである。
アニメーションは **意味のある箇所だけ** に絞り、常時動く背景や無限ループ演出は原則使わない。

技術選定:
- **React Native Reanimated** の `entering` / `exiting` / `layout` アニメーションを中心に使う
- 無限ループは LIVE・HOT など「リアルタイム性を示すバッジ」に限定する
- `react-native-gesture-handler` との組み合わせは FAB 等インタラクション系のみ

---

## 2. 優先順位

| 順位 | 対象 | 難易度 | 投資対効果 |
|------|------|--------|------------|
| 1 | FAB 開閉アニメーション | 中 | 最高 — アプリ全体の完成度が一段上がる |
| 2 | Nearby カード出現アニメーション | 低 | 高 — 情報量の多い一覧が軽やかに見える |
| 3 | ScanHeader の LIVE / HOT 脈動 | 低 | 高 — リアルタイム感が大きく増す |
| 4 | AI 検索の状態遷移アニメーション | 中 | 中 — AI画面が賢く反応している印象になる |
| 5 | タブ中央 AI アイコンのフォーカス演出 | 低 | 中 — 触って気持ちいい、主役感が出る |

---

## 3. 各提案の詳細

### 3.1 FAB 開閉アニメーション（優先度: 1）

**対象ファイル**: `components/layout/Fab.tsx`

**現状**: `useState` の `open` で表示/非表示を切り替えているだけ。Backdrop とメニューが瞬時に出現・消滅する。

**提案内容**:

1. **Backdrop**: `FadeIn` / `FadeOut` で不透明度をアニメーション
2. **メニュー2枚**: 下から `translateY` + `opacity` でスタッガー表示（1枚目 0ms、2枚目 80ms 遅延）
3. **FAB アイコン**: `PenLine` → `Plus(45deg回転)` の切り替えを `withSpring` で回転アニメーション化
4. **FAB ボタン本体**: タップ時に軽い `spring` スケール（1.0 → 0.92 → 1.0）

**実装の方向性**:

```tsx
// Backdrop
<Animated.View entering={FadeIn.duration(200)} exiting={FadeOut.duration(150)}>

// メニューアイテム
<Animated.View entering={FadeInDown.delay(index * 80).springify()}>

// FAB アイコン回転
const rotation = useSharedValue(0);
// open 切替時に withSpring(open ? 45 : 0)
const animStyle = useAnimatedStyle(() => ({
  transform: [{ rotate: `${rotation.value}deg` }],
}));
```

**注意点**:
- 現在の `open && (...)` による条件レンダリングは、Reanimated の `exiting` と相性が悪い場合がある。`Animated.View` の `exiting` を使うなら、アンマウントタイミングの制御が必要
- `zIndex: 48, 52` の既存構造は維持する

### 3.2 Nearby カード出現アニメーション（優先度: 2）

**対象ファイル**:
- `app/(tabs)/nearby.tsx` — 存在しないため新規作成 or `index.tsx` が Nearby 画面として機能している可能性あり
- `components/features/nearby/NearbyPostItem.tsx`

**現状**: `NearbyPostItem` は静的に一覧レンダリングされている。FlatList / ScrollView 内で一括表示。

**提案内容**:

各カードに `FadeInUp` を `index * 60ms` ずつ遅延させて適用する。

**実装の方向性**:

```tsx
// NearbyPostItem に entering を追加
import Animated, { FadeInUp } from "react-native-reanimated";

// Pressable を Animated.createAnimatedComponent(Pressable) に変更するか、
// 外側を Animated.View でラップする
<Animated.View entering={FadeInUp.delay(index * 60).duration(400).springify()}>
  <Pressable ...>
    {/* 既存の内容 */}
  </Pressable>
</Animated.View>
```

**注意点**:
- `index` を props として渡す必要がある（現状の `NearbyPostItemProps` に追加）
- `featured` カードは遅延なし（`delay(0)`）で先に表示すると自然
- FlatList 使用時は `initialNumToRender` との兼ね合いに注意

### 3.3 ScanHeader の LIVE / HOT 脈動（優先度: 3）

**対象ファイル**: `components/features/nearby/ScanHeader.tsx`

**現状**: LIVE バッジ（line 30-33）と HOT バッジ（line 40-42）は静的な `View` + `Text`。

**提案内容**:

LIVE と HOT バッジに、1.2〜1.8秒周期のゆるいスケール・不透明度変化を付ける。

**実装の方向性**:

```tsx
import Animated, {
  useSharedValue, useAnimatedStyle,
  withRepeat, withSequence, withTiming, Easing,
} from "react-native-reanimated";

// コンポーネント内
const pulse = useSharedValue(1);

useEffect(() => {
  pulse.value = withRepeat(
    withSequence(
      withTiming(1.08, { duration: 800, easing: Easing.inOut(Easing.ease) }),
      withTiming(1.0, { duration: 800, easing: Easing.inOut(Easing.ease) }),
    ),
    -1, // 無限リピート
    true,
  );
}, []);

const pulseStyle = useAnimatedStyle(() => ({
  transform: [{ scale: pulse.value }],
  opacity: 0.85 + (pulse.value - 1.0) * 1.5, // 1.0→0.85, 1.08→0.97
}));

// LIVE バッジに適用
<Animated.View style={[pulseStyle, { flexDirection: "row", ... }]}>
  <Eye size={12} color="#000" />
  <Text ...>LIVE</Text>
</Animated.View>
```

**注意点**:
- 変化量は控えめにする（scale 1.0〜1.08、opacity 0.85〜0.97 程度）
- LIVE と HOT で周期を微妙にずらすと機械的に見えない（LIVE: 1.6s、HOT: 1.4s など）
- パフォーマンス: Reanimated の UI スレッド実行なので JS スレッドへの負荷はほぼゼロ

### 3.4 AI 検索の状態遷移アニメーション（優先度: 4）

**対象ファイル**: `app/(tabs)/ai.tsx`

**現状**: `isSearching` による条件分岐で「通常モード」と「検索モード」を切り替え。切替は瞬時。

**提案内容**:

1. **検索バーのマイク / クリアボタン**: クロスフェードで切り替え
2. **サジェストチップ**: スタッガー表示（`FadeInRight` を index * 50ms 遅延）
3. **検索結果カード**: `FadeInUp` で表示
4. **ローディング → 結果**: `Layout` アニメーションで自然に高さ遷移

**実装の方向性**:

```tsx
// 検索結果カード
{searchResults.map((ev, i) => (
  <Animated.View key={ev.id} entering={FadeInUp.delay(i * 60).duration(350)}>
    <PulseEventCard event={ev} t={t} />
  </Animated.View>
))}

// マイク ↔ クリアボタンのクロスフェード
// Animated.View の entering={FadeIn.duration(200)} exiting={FadeOut.duration(150)}
```

**注意点**:
- `SuggestionChips` コンポーネント内部の修正も必要になる可能性がある
- debounce（300ms）との組み合わせで、アニメーション開始タイミングが不自然にならないよう調整
- `ActivityIndicator` の表示/非表示にも `FadeIn` / `FadeOut` を付けると滑らか

### 3.5 タブ中央 AI アイコンのフォーカス演出（優先度: 5）

**対象ファイル**: `app/(tabs)/_layout.tsx` (line 46-63)

**現状**: `focused` で背景色とボーダーを切り替えているが、切替は瞬時。

**提案内容**:

選択時に `scale` アニメーション（1.0 → 1.12）と、`accent` カラーの淡いグロー（`shadowOpacity` のアニメーション）を付ける。

**実装の方向性**:

```tsx
// _layout.tsx の tabBarIcon 内で
// expo-router の Tabs は tabBarIcon に Animated コンポーネントを渡せる

<Animated.View
  style={[
    styles.centerIcon,
    {
      backgroundColor: focused ? t.accent : t.surface2,
      borderColor: focused ? "transparent" : t.border,
      borderWidth: focused ? 0 : 1,
      transform: [{ scale: focused ? 1.12 : 1.0 }],
      // グロー
      shadowColor: t.accent,
      shadowOffset: { width: 0, height: 0 },
      shadowOpacity: focused ? 0.4 : 0,
      shadowRadius: focused ? 12 : 0,
    },
  ]}
>
  <Sparkles size={20} color={focused ? "#000" : t.sub} />
</Animated.View>
```

**注意点**:
- `expo-router` の `Tabs` コンポーネントが `tabBarIcon` 内の Animated を正しく扱えるか確認が必要
- タブ切替の度にリレンダーされるため、`withSpring` で滑らかにするなら `useSharedValue` + `useAnimatedStyle` を使った専用コンポーネントに切り出す方が安全
- Android では `shadowOpacity` が効かないため、`elevation` または背景のグラデーション円で代用する

## 4. やらないこと

以下は現時点では **非推奨** とする。

| 演出 | 理由 |
|------|------|
| 常時動く大きな背景アニメーション | 情報閲覧の妨げになる。NaviOs は「すばやく見る」文脈 |
| すべてのカードに無限ループ演出 | 動きすぎると読みにくさが先に来る |
| 画面遷移のカスタムトランジション | expo-router のデフォルトで十分。コスト対効果が低い |

## 5. 共通の実装ガイドライン

### 使用ライブラリ

- `react-native-reanimated` — すでにプロジェクトに導入済み
- 追加パッケージは不要

### アニメーションの数値基準

| パラメータ | 推奨値 | 備考 |
|-----------|--------|------|
| `entering` の `duration` | 300〜450ms | 速すぎると見えない、遅すぎるとモタつく |
| スタッガー遅延 | 50〜80ms / item | 5件以上は最大遅延 400ms でキャップ |
| `spring` の `damping` | 12〜18 | 低すぎるとバウンスしすぎる |
| 脈動の周期 | 1.2〜1.8s | これより速いと落ち着かない |
| scale の変化量 | 1.0〜1.12 | 大きすぎるとおもちゃっぽくなる |

### パフォーマンス注意

- Reanimated のアニメーションは UI スレッドで実行されるため JS スレッドへの負荷は低い
- ただし `entering` を FlatList の全アイテムに付けると初回レンダリングが重くなる可能性がある → `initialNumToRender` や `maxToRenderPerBatch` で制御
- 無限ループアニメーションはアンマウント時に `cancelAnimation` で止めること

## 6. この書類の扱い

本書は、アニメーション実装フェーズの設計指針としてまとめたものである。
